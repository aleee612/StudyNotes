## 为什么要把结果解析成 JSON？

因为我们发出的请求：

```
fetch(URL)
```

得到的结果是一个 **Response 对象**，这个对象**本质上是对 HTTP 响应的封装**，而不是你真正要的数据。

你需要用 `.json()` 把它“解开”，才能拿到我们常用的 JavaScript 对象（Object 或 Array）。

---

## 网络请求结果返回的到底是什么？

当你用 `fetch` 请求一个 API，浏览器收到的东西本质上是：

```
HTTP/1.1 200 OK
Content-Type: application/json

{
  "data": {
    "channels": [
      { "id": 1, "name": "推荐" },
      { "id": 2, "name": "科技" },
      ...
    ]
  }
}
```

这段看起来像 JavaScript 对象的内容，其实是 **纯文本（字符串）格式的 JSON**。

所以：

- **网络层返回的是文本**（字符串），比如：

```
"{\"data\":{\"channels\":[{\"id\":1,\"name\":\"推荐\"}]}}"
```

- `res.json()` 会把这段字符串转成你能用的对象：

```
{
  data: {
    channels: [
      { id: 1, name: "推荐" }
    ]
  }
}
```

---

## 为什么叫 JSON？

JSON 全称是：**JavaScript Object Notation（JavaScript 对象表示法）**

也就是说，**API 返回的数据格式，是为了让前端能方便地拿来用**，所以用 JSON 是最常见最主流的方式。

---

## 除了 `.json()`，还有别的解析方法吗？

当然有，看这个表格👇：

|   |   |
|---|---|
|方法|作用|
|`res.json()`|把返回结果解析为 JSON 对象|
|`res.text()`|把返回结果解析为纯文本字符串|
|`res.blob()`|把返回结果解析为二进制 Blob 对象|
|`res.arrayBuffer()`|把返回结果解析为 ArrayBuffer（处理音频/视频）|
|`res.formData()`|把返回结果解析为表单数据|

---

## 总结

你之所以要用 `.json()`，是因为：

服务器返回了“JSON 格式的字符串”，而你要的是“可以直接用的数据对象”。
