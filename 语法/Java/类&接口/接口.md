### 语法
#### 定义接口 interface
```java
public interface Comparable{
	int compareTo(Object other);
}
```
#### 类实现接口 implements
```java
class Emp implements Comparable{
	public int compareTo(Emp other){
		...
	}
	...
}
```
### 作用
- 确保所有的实现该接口的类都需要包含其中的方法/字段
- 需注意，在接口定义方法的同时也定义了方法的返回值
### 特性
- 所有方法都自动为public，所有字段都默认为public static final
- 接口可以定义常量
- 接口绝不会有**实例字段**
- 接口中的字段 / 方法是不初始化的，留给后续的类实现（java8之前），而在Java8之后逐渐开始提供简单方法。但这些方法依旧不能引用实例字段
- 在类中实现接口方法需声明public
### 属性
#### 声明与定义
- 接口无法new，但能声明变量
```java
Comparable x;
```
- 这个变量必须引用实现了这个接口的类的对象
```java
x = new Emp(...);
```
- 可以使用instanceof检查一个对象是否实现了某个特定的接口
```java
if(Obj instanceof Comparable) { ... }
```
#### 接口扩展extends
```java
public interface Power extends Moveable{
	double min();
}
```
### 方法类型
#### 静态方法
```java
public interface Path{
	public static Path of(String first,String... more)
	{ ... }
}
```
- 有违设计理念，用的不多
#### 私有方法
- 私有方法只能在接口本身中使用，用法有限，只能作为接口中其他方法的辅助方法
#### 默认方法
```java
public interface Comp<T>
{
	default int compareTo(T other){
		return 0;
	}
}
```
- 必须用default标记
- 可以在实现该接口的类中不实现该方法，直接使用默认
- 当类继承的多个接口中有相同的默认方法而在类中不覆盖方法时，将发生冲突；此时需要指定使用哪个方法
```java
class Student implements Person,Named
{
	public String GetName(){
		return Person.super.GetName();
	}
}
```
- 若两个接口中没有为共享方法提供默认实现，则不会报错（因为必须在子类中实现，或子类本身为抽象类）
- 当类继承的父类和接口存在相同方法时，将只会使用父类方法（类优先）
