### 命中率
**缓存命中**：缓存中存在需要的数据
#### 命中率计算
$H$：命中率
$N_c$：命中次数
$N_m$：访问主存次数（即未命中次数）
$$H=N_c/(N_c+N_m)$$
#### Cache-主存系统平均访问时间
$T_c$：访问Cache的时间（命中所需时间）
$T_m$：主存调入一个块的时间
$T_c+T_m$：未命中所需时间
$T_a$：平均访问时间
$$T_a=HT_c+(1-H)·(T_m+T_c)=T_c+(1-H)·T_m$$

### 组成
| 部件              | 作用                          |
| --------------- | --------------------------- |
| cache block 缓存块 | 存数据                         |
| cache line      | Tag，Flags，缓存块的统称            |
| Tag             | 和 地址索引index 一起使用，用于判断缓存是否命中 |
#### Cache块大小计算
$$\text{块内地址位数} = \log_2(\text{每个 Cache 块包含的字节数})$$
#### 主存与cache的映射
| 映射方式  | 计算方法                                    | 主缓存关系                    | 地址结构              |
| ----- | --------------------------------------- | ------------------------ | ----------------- |
| 直接映射  | $Cache块号 = 主存块号 \% 缓存中的总块数$             | 主存块只能映射到缓存中的**一个特定**缓存块  | $标记+Cache行号+块内地址$ |
| 全相联映射 | $Cache行 = 任意行$                          | 主存中的任何块可映射到缓存中的**任意**缓存块 | $标记+块内地址$         |
| 组相联映射 | $组号 = 主存块号\%组数$ <br>$Cache行=组内N行中的任意一个$ | 缓存块分组，主存块可映射到组中的任意一个缓存块  | $标记+组号+块内地址$      |
- N 路组相联表示一个 **组中有 N 个 cache 块**，而不是 cache 中一共有 N 个组
- 路数计算方式：
$$\text{路数 (n)} = \frac{\text{Cache 总容量}}{\text{块大小 (Block Size)} \times \text{组数 (Sets)}}$$

$$\text{路数 (n)} = \frac{\text{Cache 总行数}}{\text{组数 (Sets)}}$$
- 关联度：全相联 > 组相联 > 直接映射
- 关联度高，**缓存冲突** 减少，**命中率** 提高，硬件开销 和 访问延迟 增加
**块内地址：**
- 确定访问数据在一个 主存块 / cache 块 内的具体偏移
**Cache 块匹配字段：**
- 确定该主存块对应 cache 块位置
**标记：**
- 通过比较 tag 判断是否真正命中
- Cache 标记字段的位数 = $\log_2(\frac{\text{主存块数}}{\text{Cache组数}})$

### cache 存储结构
#### 容量计算
$$Cache总容量=(每行标记位数+每行数据位数)·Cache总行数$$
#### 存储结构
|      | 有效位      | 脏位                 | LRU位（仅LRU算法时存在） | 标记位                   | 数据块               |
| ---- | -------- | ------------------ | --------------- | --------------------- | ----------------- |
| 位数   | 1位       | 1位                 | $log_2(组内块数)$   | -                     | -                 |
| 作用对象 | Cache行   | 数据                 | LRU替换位          | 与 物理地址 的 **tag字段** 匹配 | -                 |
| 作用   | 是否存储有效数据 | 标记数据是否被修改( 回写法 需要) | 控制计数器更新，块替换     | 判断是否命中                | 缓存的主存块副本，是实际存储的数据 |

### 块替换
|      | 随机RAND           | 先入先出FIFO    | 最近最少使用LRU            | 不经常使用LFU              |
| ---- | ---------------- | ----------- | -------------------- | --------------------- |
| 替换方式 | 随机选择一个Cache行进行替换 | 最早装入的Cache行 | 优先替换最近最久未被访问 的Cache行 | 换一段时间内累计访问次数最少的Cache行 |
| 优点   | 实现简单             | 实现较容易       | 平均命中率通常高于FIFO        |                       |
| 缺点   | 命中率通常较低          | 命中率不高       |                      |                       |
- 在 **权相连**映射 和 **组相连**映射中，有块替换算法的需要，用于判断应该替换哪个块
- 整体与[[虚拟内存管理#页置换算法|页面置换算法]] 差不多
#### LRU细拆
1. 为每组Cache 维护一组计数器（LRU替换位），记录各Cache行的相对访问顺序
2. 计数器更新规则：
	1. 命中 -> 所命中行的计数器清零，比其低的计数器**加1**，其余不变
	2. 未命中且**有**空闲行 -> 新装入的行的计数器**置0**，其他非空闲行全加1
	3. 未命中且**无**空闲行 -> 替换计数值最大的行，新装入的行的计数器**置0**，其余全加1
