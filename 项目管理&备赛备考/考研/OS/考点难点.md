### 内存
#### 连续分配管理方式
|      | 单一连续分配                    | 固定分区分配                                         | [[内存管理概念#分配算法\|动态分区分配]]       |
| ---- | ------------------------- | ---------------------------------------------- | ----------------------------- |
| 分配方式 | 内存被划分为系统区和用户区             | 将用户内存空间划分为若干**大小固定**的分区，每个分区仅能容纳一道作业，分区大小可互不相等 | 按其实际需求动态分配一块**大小恰好匹配**的连续内存空间 |
| 优点   | 结构简单、无外部碎片，无须内存保护机制       | 无外部碎片                                          |                               |
| 缺点   | 仅适用于单用户、单任务的操作系统，且内存利用率极低 | 无法支持多个进程共享同一内存区域，内存利用率较低                       | 有外部碎片，开销较大                    |
| 内部碎片 | 存在                        | 存在                                             | -                             |
#### 重定位
**静态重定位：**
- 在程序装入内存之前，把程序中的逻辑地址（通常从0开始）直接改写成确定的物理地址
**动态重定位：**
- 程序运行时，每次访问地址都“现场转换”
- 系统借助一个**重定位寄存器**用于存放该模块在内存中的起始物理地址
- 支持**程序**在内存中**移动**，便于操作系统进行紧凑
