作用：在磁盘之上构建 **逻辑结构**，以 盘块（或簇）为单位进行空间分配和操作
### 文件系统结构
| 部件        | 位置       | 作用                                                                |
| --------- | -------- | ----------------------------------------------------------------- |
| 引导区 *MBR* | 磁盘的第一个扇区 | 包含引导加载程序，负责**启动**计算机 并 加载操作系统**内核**                               |
| 分区表       | 第一个扇区之后  | 记录磁盘上的 **分区信息**（分区起始位置、大小和文件系统类型）                                 |
| 分区        |          | 存储有文件系统（包含元数据和数据区）                                                |
| 元数据       | 分区内      | 超级块：分区的块的数量、块的大小、空闲空间的管理方式<br>空闲块信息：管理空闲块的存储数据<br>inodes：管理文件的元数据 |
| 数据区       | 分区内      | 存储实际的文件数据（内容）                                                     |
### 外存空闲管理
#### 空闲表法
- 数据结构：链表
	1. 链表的头节点保存第一个空闲块的位置
	2. 每个节点的 **next** 指针指向 **下一个空闲块**，形成一条从头到尾依次链接的空闲块序列
#### 位图法
- 数据结构：二进制数组
	1. 位图中的每一位对应磁盘上一个固定大小的数据块，通过该位的取值即可快速判断该块的使用状态
	2. 使用 0, 1 判断是否被使用
#### 成组链接法
- 数据结构：数组+链表
	1. 使用一个磁盘块来保存若干空闲盘块号
	2. 通过链表把这些块组织起来
### 虚拟文件系统 VFS
- 屏蔽不同存储介质（本地磁盘、网络共享、光盘等）和各类文件系统的实现细节
- 相当于硬件层面的封装
### 文件系统挂载
- 把一个独立的文件系统与文件树关联起来，使得该文件系统中的文件和目录能够被正常访问和管理
- 步骤：
	1. 识别存储设备  
	2. 选择挂载点 ：挂载点本身必须是一个已存在且为空的目录
	3. 执行挂载操作：OS调用相应的挂载函数，将外部文件系统的根目录映射到挂载点
	4. 管理与维护
