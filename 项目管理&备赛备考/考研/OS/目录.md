### 目录
- 形状：树形
- 本质：文件
- 存储结构：目录项（包含文件名和inode）
### 软连接与硬链接
#### 硬链接
本质：
- 为同一个 索引节点（inode）**创建多个目录项**
- 即 inode : 文件名 为 1 : n
创建方式：
```bash
ln file1 file2
```
特点：
- 共享同一个 inode 号
- 不会创建数据副本
- 不能用于**目录**
- 仅限于同一个文件系统内部，不能跨越挂载点或磁盘分区
#### 软链接
本质：
- 包含 指向目标文件路径的字符串 的文件
创建方式：
```bash
ln -s file1 file2
```
特点：
- 有独立文件名和 inode
- 删除原文件后，软链接不会失效为独立文件，但会变成 **悬挂链接**，无法再访问目标内容
- 可跨文件系统使用，可使用于目录
