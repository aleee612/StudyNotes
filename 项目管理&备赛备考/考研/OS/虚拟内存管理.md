[[Cache]]
#### 驻留集
- 某个进程在执行过程中，当前实际存放在物理内存中的页面集合
**RSS**：
- 驻留集大小，以页框数量表示
- 过小：缺页中断
- 过大：挤压其他进程生存空间

---
### 页面大小
| 影响维度        | 页面较小                         | 页面较大                            |
| ----------- | ---------------------------- | ------------------------------- |
| 内存利用率       | **高**<br>页内碎片平均较小，浪费少        | **低**<br>页内碎片平均较大，浪费较多          |
| 页表长度/开销     | **大**<br>需要更多页表项，可能需要多级页表    | **小**<br>页表项少，节省页表占用的内存         |
| TLB (快表) 效率 | **较低**<br>覆盖内存范围小，命中率相对较低    | **较高**<br>同样的 TLB 条目覆盖更多内存，命中率高 |
| 缺页中断频率      | **较高**<br>单页内容少，程序运行容易跨页触发中断 | **较低**<br>单页容纳内容多，局部性原理发挥更好     |
| I/O 传输效率    | **低**<br>频繁读写磁盘小块数据，寻道开销大    | **高**<br>单次 I/O 交换数据量大，传输效率高    |
| 程序加载速度      | **较慢**<br>初始启动时需要多次调页        | **较快**<br>较少次数的 I/O 即可加载常用代码    |

### 页置换算法
| 算法    | 中文名       | 核心思想      | 优点                            | 缺点                                                         |
| ----- | --------- | --------- | ----------------------------- | ---------------------------------------------------------- |
| OPT   | 最优        | 未来最长不用    | • 缺页率最低<br>• 可作为理论性能上界        | • 需要预知未来访问序列<br>• 实际无法实现                                   |
| FIFO  | 先进先出      | 最早进来的先滚   | • 实现最简单<br>• 开销小              | • 可能产生 Belady 现象（增加进程分配的物理块，缺页次数反而可能上升）<br>• 不考虑访问频率和时间局部性 |
| LRU   | 最近最久未使用   | 最近最久没用    | • 符合程序局部性原理<br>• 性能接近最优       | • 实现复杂<br>• 需要额外硬件或较高维护成本                                  |
| Clock | 时钟（近似LRU） | LRU 低成本近似 | • 实现简单<br>• 性能接近LRU<br>• 开销适中 | • 只是LRU近似<br>• 极端情况下效果下降                                   |
| LFU   | 最少使用      | 用得最少的滚    | • 适合稳定访问模式<br>• 能保留热点页面       | • 早期热点可能长期占用内存<br>• 需要维护访问计数                               |
- clock改进：根据页面的（访问位,，修改位），优先级 访问位 > 修改位

---
### 页表项
|**字段名称**|**别名**|**核心作用**|**触发后果**|
|---|---|---|---|
|**有效位**|状态位 / P位|该页是否在内存中|为 0 时触发**缺页中断**|
|**修改位**|脏位 / M位|该页调入后是否被修改过|为 1 时页面换出需写回磁盘|
|**访问位**|引用位 / A位|近期是否被访问过|为 **页面置换算法**（如LRU）提供依据|
|**保护位**|存取控制位|读/写/执行权限控制|违规访问触发权限异常|

### 置换方法
| 维度      | 概念名称          | 关注点                              |
| ------- | ------------- | -------------------------------- |
| 分配/选择策略 | 局部置换 / 全局置换   | 谁该倒霉？（是从我自己的块里挑一个，还是从全系统的块里挑一个？） |
| 数据流向策略  | 进程内置换 / 进程外置换 | 去哪儿待着？（是换到内存的其他位置，还是彻底踢出内存关进硬盘？） |
#### 分配策略
| 维度     | 局部置换                     | 全局置换                          |
| ------ | ------------------------ | ----------------------------- |
| 置换范围   | 只能从**当前进程**自己的物理块中选一页换出  | 可以从**整个内存**的空闲块或**其他进程**的块中换出 |
| 物理块数变化 | 进程分配到的物理块总数保持不变          | 进程分配到的物态增理块总数会动减              |
| 独立性    | 高<br>进程的缺页率只受自身行为影响      | 低<br>一个进程频繁缺页会抢占其他进程的块        |
| 内存利用率  | 相对较低<br>（某些进程块不够用，某些又闲置） | 较高<br>（内存资源全系统动态调度）           |
| 系统稳定性  | 强<br>能有效抑制抖动的蔓延          | 弱<br>容易产生连锁反应，导致系统整体效率下降      |
| 常见组合   | **固定分配** + 局部置换          | **可变分配** + 全局置换（最常用）          |
