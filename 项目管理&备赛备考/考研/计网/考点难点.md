## 物理层
### 调制
| 调制类别  | 调制名称   | 英文缩写   | 改变的物理量      | 状态数 (M) | 每码元比特数 (n) | 核心要点                                 |
| ----- | ------ | ------ | ----------- | ------- | ---------- | ------------------------------------ |
| 基础二进制 | 幅移键控   | ASK    | 振幅          | 2       | 1 bit      | 容易受噪声干扰，类似于“开/关”信号。                  |
|       | 频移键控   | FSK    | 频率          | 2       | 1 bit      | *唯一* 需要两个不同频率载波的二进制方式。               |
|       | 相移键控   | PSK    | 相位          | 2       | 1 bit      | 抗干扰能力优于 ASK 和 FSK。                   |
|       | 差分相移键控 | DPSK   | 相对相位        | 2       | 1 bit      | 利用前后码元的相位差表示数据。                      |
| 多元/混合 | 正交相移键控 | QPSK   | 相位          | 4       | 2 bit      | 拥有 4 种相位状态（如 45°, 135°, 225°, 315°）。 |
|       | 8PSK   | 8PSK   | 相位          | 8       | 3 bit      | 拥有 8 种相位状态。                          |
|       | 正交振幅调制 | 16-QAM | **振幅 + 相位** | 16      | 4 bit      | 混合调制，极大提高频谱利用率。                      |
|       | 正交振幅调制 | 64-QAM | **振幅 + 相位** | 64      | 6 bit      | 常用于高速 WiFi 和 移动网络（如 4G/5G）。          |
### 物理层接口特性
- 机械特性：接口长什么样？
- 电气特性：接口电伏多少？速率多少？
- 功能特性：这条线干啥用？
- 过程/规则特性：啥时候做事？
## 数据链路层
### 主要功能
1. 帧定界（定义数据格式）
2. 差错检测&纠错
3. 流量控制（不是拥塞控制，拥塞是防止过多分组注入网络导致网络性能下降，是网络层的事）
4. 传输透明
5. 可靠传输
### 差错检测 & 纠错
#### CRC
1. 原数据 M 位
2. G(x)， r+1 位
3. FCS生成方式：步骤 `2` 对 `1+r*0` 做模二除法运算（即依次按位异或，不足的向后补位，直至不够为止），余数为FCS（`FCS的位数==r`）
4. 发送：原数据 + FCS
5. 校验：用相同的 G(x) 对接收到的帧进行模二除法运算，余数为0则为无差错
#### 海明码
1. 原数据 n 位
2. 校验 k 位
3. $2^k≥n+k+1$ 
4. 若码距为 d，则可校验出错位数 d - 1 位，可纠正位数 $\dfrac{d-1}{2}$ 位
