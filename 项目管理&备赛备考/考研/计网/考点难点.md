> [!warning]+ 做题须知
> 区分两种 MB：
> 
> **计算机存储容量的 MB（二进制）**：
   1 MB = $2^{20}$ Byte = 1,048,576 Byte
   常用于 **内存、ROM、显存、硬盘理论容量**（尤其在计组和 MAR 题里）
   >
   **数据传输速率的 MB（十进制）**：
   1 MB = $10^{6}$ Byte = 1,000,000 Byte
   常用于 **带宽 / 硬盘标称速度 / 网络速度**
   Mbps、MB/s 里往往默认十进制（工程计量单位）

> [!note] 服务访问点
>数据链路层：帧的“类型”字段
>网络层：IP数据报的“协议”字段
>传输层：“端口号”字段
>应用层：“用户界面”

> [!warning] 常用首部长度
> MAC：18B
> IP：最大60B，最小20B
> TCP：20B
> UDP：8B

### 数据链路层
#### 主要功能
1. 帧定界（定义数据格式）
2. 差错**检测** & **纠错** 
3. 流量控制（不是拥塞控制，拥塞是防止过多分组注入网络导致网络性能下降，是网络层的事）
4. 传输透明（主要体现在交换机）
5. 可靠传输
- [[差错检测和纠正]] （CRC，海明码）
- [[可靠传输]] （停等，GBN，SR，性质和计算）
- [[介质访问控制与算法]] （尤其CDMA，CSMA）
- [[局域网与广域网]] （几种典型局域网，ppp，透明传输）

###### 各设备转发速度比较
| 设备            | 工作层次  | 转发依据     | 转发速度      |
| ------------- | ----- | -------- | --------- |
| 集线器（Hub）      | 物理层   | 广播发送     | 最快（无选择转发） |
| 网桥（Bridge）    | 数据链路层 | MAC地址    | 快         |
| 二层交换机（Switch） | 数据链路层 | MAC地址表   | 很快        |
| 路由器（Router）   | 网络层   | IP地址、路由表 | 较慢        |
| 网关（Gateway）   | 高层协议  | 协议转换     | 最慢        |
### 传输层
#### 主要功能
1. 提供主机间**进程**（端到端）的逻辑通信
2. 能提供可靠的服务
3. 复用和分用：
	复用：发送方，多个应用进程可使用同一个传输层协议传送数据
	分用：接收方，剥去报文首部后，能将数据正确交付给对应的目的应用进程 
4. 差错检测
	针对整个报文：TCP重传，UDP丢弃
5. 提供面向连接和无连接的服务
