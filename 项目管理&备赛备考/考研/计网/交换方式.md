### 传输时间计算
- **发送时间 = 数据大小 ÷ 带宽**
- **传播延迟 = 距离 ÷ 速度**

| 单位        | 换算公式                                                     | 说明              |
| --------- | -------------------------------------------------------- | --------------- |
| bps（比特/秒） | 基本单位                                                     | 1 bps = 1 bit/s |
| Kbps      | $(1 \text{Kbps} = 10^3 \text{bps} = 1000\ \text{bps})$   | 千比特每秒           |
| Byte/s    | $(1\ \text{B/s} = 8\ \text{bps})$                        | 8 bit = 1 Byte  |
| KB/s      | $(1\ \text{KB/s} = 1000\ \text{B/s} = 8000\ \text{bps})$ | 千字节每秒           |
##### 以太网帧大小（经典 Ethernet II）
- 以太网帧长度 **64B ~ 1518B（不含前导码）**，  
- **数据字段 46~1500B**。

- 当两个需要通讯的设备之间有三段转发路径
##### 分组交换
总传输时间 = 总数据量 / 带宽 + 一次传输数据量（一个分组大小） / 带宽
##### 报文交换
总传输时间 = 一次传输数据量（一个分组报文大小）* 报文数 / 带宽
##### 最大吞吐量 = 瓶颈链路带宽
$Throughputmax⁡=min⁡(R1,R2,…,Rn)$   
### 虚电路与数据报

| 特性     | 数据报                | 虚电路             |
| ------ | ------------------ | --------------- |
| 连接性    | 无连接，每个分组独立处理       | 有连接，建立虚电路后传输分组  |
| 路由     | 每个分组独立路由           | 虚电路建立后，沿着相同路径路由 |
| 通信开销   | 较低，不需要建立和维护连接      | 较高，建立连接需要额外开销   |
| 分组到达顺序 | 不保证分组到达的顺序         | 保证分组按照发送顺序到达    |
| 分组丢失   | 可能出现分组丢失，需要上层协议处理  | 较低的丢包率，可靠传输     |
| 带宽利用率  | 不需要预留带宽，按需使用       | 预留带宽，可能造成资源浪费   |

### 错题
> 对于 100 Mbps 的以太网交换机，当输出端口无排队，以直通交换 (cut-through switching) 方式转发一个以太网帧（不包括前导码）时，引入的转发延迟至少是（ ）。
	A.0 μs       B.0.48 μs       C.5.12 μs       D.121.44 μs
- 直通交换属于分组交换中的一种交换方式（以太网交换机的数据帧转发方式）
- 直通交换方式只检查帧的目的地址，共 6B
- 传输延迟 6 × 8bit / 100Mbps = 0.48μs
