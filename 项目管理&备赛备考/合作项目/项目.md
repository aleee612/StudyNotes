| **1.2 数据库设计**                                                     | 设计用户表、会议室表的基础结构。                | *无*                                                        | **核心设计：** 用户表 (`id`, `username`, `password_hash`)、会议室表 (`id`, `name`, `owner_id`, `created_at`)。 |
| **1.3 注册/登录**                                                     | 实现用户注册和登录功能。                    | **表单验证 (Validation)**，发送 **HTTP POST 请求**，接收并存储 **Token**。 | **API 接口：** 设计 `/register` 和 `/login` 接口，处理密码哈希，生成并返回 **JWT (JSON Web Token)**。                  |

### 里程碑 3：实时通信核心（WebSocket 驱动）

这个阶段引入核心的 WebSocket 技术，实现实时连接。

| 步骤 | 任务描述 | 前端侧重点 (她) | 后端侧重点 (您) |
| :--- | :--- | :--- | :--- |
| **3.1 WebSocket 架构** | 选定和部署 WebSocket 解决方案。 | **WebSocket 客户端连接**：处理连接、断开、重连逻辑，**全局状态**存储连接状态。 | **WebSocket 服务器搭建：** 选用支持 WebSocket 的库（例如：Node.js 的 `ws` 或 `socket.io`，或 Go 的 `Gorilla Websocket`），并设计**实时事件的通信协议**。 |
| **3.2 进入/离开房间** | 实现用户通过 WebSocket **进入特定房间**和**离开房间**的逻辑。 | **发送 `JOIN_ROOM` 事件** (`room_id`, `token`)，监听 `USER_JOINED` / `USER_LEFT` 事件，并**实时更新房间用户列表**。 | **连接池管理：** 维护一个**用户 ID 到 WebSocket 连接**的映射表，以及**房间 ID 到用户 ID 列表**的映射表。验证 Token 后，将连接添加到对应的房间组。 |
| **3.3 实时发送/接收消息** | 实现用户在房间内发送一条简单的文字消息，并能让房间内所有用户实时接收。 | **发送 `SEND_MESSAGE` 事件** (`room_id`, `content`)，**监听 `RECEIVE_MESSAGE` 事件**，并将消息实时渲染到页面。 | 接收 `SEND_MESSAGE` 事件，验证用户权限后，通过**房间连接池**将消息**广播 (Broadcast)** 给房间内所有其他用户。 |
| **🏁 里程碑 3 成果：** 用户可以进入房间，实时看到其他用户进出，并能实时发送/接收简单消息。 |
| **搜索关键词：** `[您的后端语言/框架] WebSocket 教程`，`Socket.IO 房间广播`，`WebSocket 客户端实现`。 |

### 里程碑 4：弹幕功能与高性能渲染（高难度挑战）

这个阶段是项目的灵魂，重点是弹幕的**高性能渲染**。

| 步骤                                                                    | 任务描述                              | 前端侧重点 (她)                                                                                    | 后端侧重点 (您)                                                                                                        |
| :-------------------------------------------------------------------- | :-------------------------------- | :------------------------------------------------------------------------------------------- | :--------------------------------------------------------------------------------------------------------------- |
| **4.1 弹幕数据库**                                                         | 设计弹幕存储结构。                         | *无*                                                                                          | **核心设计：** 弹幕表 (`id`, `room_id`, `user_id`, `content`, `time_offset`, `created_at`)。`time_offset` 记录弹幕相对于视频开头的秒数。 |
| **4.2 弹幕发送/存储**                                                       | 将实时消息升级为弹幕，并实现持久化。                | **弹幕输入框**设计，将**当前视频时间**和弹幕内容一起发送到后端。                                                         | **API/WS 处理：** 接收弹幕数据，实时广播的同时**异步存储**到数据库。                                                                       |
| **4.3 历史弹幕获取**                                                        | 实现用户进入房间后，加载已有的历史弹幕。              | **API 调用：** 页面加载时请求 `/rooms/:id/danmaku` (GET) 接口获取历史弹幕数据。                                   | **API 接口：** 设计 `/rooms/:id/danmaku` (GET) 接口，根据 `room_id` 和**时间范围**返回弹幕列表。                                       |
| **4.4 弹幕高性能渲染**                                                       | **核心挑战：** 实现**平滑滚动、无碰撞、动画**的弹幕渲染。 | **Canvas 或 CSS 动画：** 学习使用 **Canvas** 或高度优化的 **CSS Animation** 来渲染大量弹幕，实现**弹幕轨迹计算**和**轨道分配**。 | **后端优化：** 确保弹幕查询接口的**查询速度**（例如：为 `room_id` 和 `time_offset` 字段建立联合索引）。                                            |
| **🏁 里程碑 4 成果：** 弹幕功能完整实现，具备高性能的渲染效果，这是项目的最大亮点。                       |                                   |                                                                                              |                                                                                                                  |
| **搜索关键词：** `JavaScript 弹幕实现`，`Canvas 弹幕渲染`，`CSS 动画优化`，`[数据库] 联合索引优化`。 |                                   |                                                                                              |                                                                                                                  |
