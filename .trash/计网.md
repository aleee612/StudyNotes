#### 1. 网络协议的三要素及其功能

- **语法**：数据与控制信息的结构或格式。确定“怎么写”。
    
- **语义**：需要发出何种控制信息，完成何种动作以及做出何种响应。确定“是什么意思”。
    
- **同步**（或时序）：事件实现顺序的详细说明。确定“什么时候做”。
    

#### 2. 网络层两个层面的功能

- **数据层面**（转发）：路由器根据本地转发表，将收到的分组从合适的接口转发出去。处理的是单个分组。
   
- **控制层面**（路由选择）：通过路由算法计算从源到目的地的路径，生成并维护路由表。处理的是网络整体拓扑。
    

#### 3. 计算机网络中的时延（4种）

- **发送时延**：主机或路由器发送数据帧所需要的时间。$公式：数据帧长度 / 发送速率$。
    
- **传播时延**：电磁波在信道中传播一定距离所需的时间。$公式：信道长度 / 传播速率$。
    
- **处理时延**：主机或路由器在收到分组时进行首部检查、提取地址等处理所花费的时间。
    
- **排队时延**：分组在进入路由器后，在输入/输出队列中等待处理或转发的时间。
    

#### 4. 网络设备所属层次

- **集线器 (Hub)**：物理层（第1层）。
    
- **交换机 (Switch)**：数据链路层（第2层）。
    
- **路由器 (Router)**：网络层（第3层）。

### 二、 名词解释（简练背诵版）

- **计算机网络**：将地理位置不同的、具有独立功能的多个计算机系统，通过通信设备和线路连接起来，由网络软件实现资源共享和信息传递的系统。
    
- **RIP** ：路由信息协议。一种基于距离向量算法的内部网关协议，通过跳数来衡量到达目的地的路由距离。
    
- **OSPF (Open Shortest Path First)**：开放最短路径优先。一种基于**链路状态算法**的内部网关协议，使用 Dijkstra 算法计算最短路径树。
    
- **发送窗口**：在滑动窗口协议中，发送方在未收到确认的情况下，允许连续发送的数据包的最大序列号范围。
    
- **IP**，解释为：网际协议，负责在网络间寻址和路由分组。
- **ARP**，解释为：地址解析协议，将 IP 地址转换为 MAC 地址。
    
- **停等协议**：最简单的可靠传输协议。发送方每发送一个分组就停止发送，等待接收方确认，收到确认后再发送下一个分组。
    
- **超时重传**：运输层协议（如TCP）的一种机制，若发送方在规定时间内未收到接收方的确认，则认为数据丢失，重新发送该数据。
    
- **HTML**：超文本标记语言。用于创建网页的标准标记语言，通过标签定义网页的内容和结构。
    
- **流量控制**：控制发送方的发送速率，使其不要太快，以确保接收方来得及接收，防止缓冲区溢出。
