# Docker

## 容器镜像

容器的概念和VM对比就行

VM是一个硬件层的抽象，虚拟出一套硬件，运行一个完整的操作系统

容器是一个应用层的抽象，不虚拟硬件，只虚拟操作系统，所以容器之间是共享一个操作系统资源的

## Docker

### 核心概念

镜像：容器的**只读的模版**，类似光盘镜像ISO或者Java中的类

容器：镜像的一个**运行实例**，类似安装完ISO的正在运行的操作系统或者Java中的对象

仓库：集中存放镜像文件的地方
