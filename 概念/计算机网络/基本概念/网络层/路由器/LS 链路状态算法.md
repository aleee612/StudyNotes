> 考点一句话：**LS = 泛洪链路状态 + 每个路由器掌握全网拓扑 + Dijkstra 算最短路径树**

---
### 必记关键词
- **链路状态**：本路由器和邻居之间的连接关系、链路开销
- **泛洪**：收到链路状态信息后，继续转发给其他邻居，使信息扩散到全网
- **全局拓扑**：每个路由器最终都知道完整网络图
- **Dijkstra**：每个路由器以自己为源点计算最短路径树
- **典型协议**：[[OSPF开放最短路优先]]

---
### 工作流程
1. 发现邻居，测量到邻居的链路开销
2. 生成自己的链路状态信息
3. 通过 **泛洪** 发送给全网其他路由器
4. 收集所有路由器的链路状态，形成完整拓扑图
5. 运行 **Dijkstra** 算法，生成路由表

### 易考点
1. LS 属于 [[路由选择算法#按掌握的信息分类|集中式路由选择算法]]
2. LS 使用 **泛洪** 传播链路状态信息
3. LS 中每个路由器都有完整拓扑，但 **各自独立计算** 路由表
4. LS 使用 **Dijkstra**，不是 Bellman-Ford
5. LS 收敛通常比 [[DV 距离向量算法|DV]] 快，但需要维护拓扑数据库，开销更大

---
### 和 DV 的关键区别
| 对比 | LS | DV |
| --- | --- | --- |
| 交换内容 | 链路状态 | 距离向量 |
| 传播方式 | 泛洪 | 只发给邻居 |
| 掌握信息 | 全网拓扑 | 邻居给出的距离 |
| 算法 | Dijkstra | Bellman-Ford |
| 典型协议 | [[OSPF开放最短路优先]] | [[RIP 路由信息协议]] |
