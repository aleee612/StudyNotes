> 一种基于 [[LS 链路状态算法|链路状态]] 的内部网关协议
> 主要用于一个 [[AS 自治系统|AS]] 内部的中大型网络

---
### 基本机制
OSPF 交换的不是完整路由表，而是 **链路状态信息**。

核心对象：
- **LSA**：链路状态通告，描述某台路由器的邻居、链路类型、cost 等信息
- **LSDB**：链路状态数据库，保存收到的有效 LSA
- **SPF**：以自己为根运行 Dijkstra，计算最短路径树

---
### 工作流程
1. 通过 Hello 报文发现并维持邻居关系
2. 邻接路由器之间同步 LSA
3. LSA 在同一区域内洪泛
4. 各路由器形成一致的 LSDB
5. 每台路由器独立运行 Dijkstra，生成路由表

---
### 代价 cost
OSPF 使用 **cost** 作为度量值

- cost 可由管理员配置，也可按带宽计算
- OSPF 只选择总代价最小的路径
- 多条总代价相同的路径可以做等价负载均衡

---
### 分层区域
OSPF 可以把一个 AS 划分为多个区域，减少 LSA 洪泛范围和 SPF 计算压力

| 项目              | 内容                    | 作用/特点                         |
| --------------- | --------------------- | ----------------------------- |
| 分层目的            | 将一个 AS 划分为多个区域        | 减少 LSA 洪泛范围，降低 SPF 计算量，提高可扩展性 |
| Area 0（骨干区域）    | OSPF 的核心区域            | 所有区域间通信必须经过 Area 0            |
| 普通区域            | Area 0 之外的区域          | 只维护本区域的完整拓扑                   |
| 区域内部路由          | 同一区域内路由器交换链路状态信息      | 每个路由器运行 SPF 算法计算最短路径          |
| ABR（区域边界路由器）    | 连接多个区域，且至少连接一个 Area 0 | 交换、汇总区域间路由，限制 LSA 洪泛范围        |
| ASBR（自治系统边界路由器） | 连接 OSPF AS 与外部 AS     | 将外部路由引入 OSPF                  |
| 区域内传播           | LSA 在本区域内洪泛           | 不会扩散到整个 AS                    |
| 区域间通信           | 不同区域间交换路由信息           | 必须通过骨干区域 Area 0               |
| 分层优点            | 控制链路状态数据库规模           | 降低计算和存储开销，提高网络稳定性             |

### 报文与特点
- 直接封装在 [[概念/计算机网络/基本概念/网络层/IP/IP|IP]] 中，协议号为 **89**
- 支持鉴别
- 收敛较快
- 适合 AS 内部中大型网络

与 RIP、BGP 的整体对比见 [[路由选择算法#常见协议定位]]。
