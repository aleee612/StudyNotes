### 描述路由问题
> 在带权图中，为分组选择一条从源结点到目的结点的最低开销路径

![[Pasted image 20251022172535.png]]

- 网络表示为 `G=(N,E)`
- `N`：结点集合
- `E`：边集合
- `c(x,y)`：结点 `x` 与 `y` 之间边的开销
- 若 `x` 与 `y` 不直接相连，则 `c(x,y)=∞`
- 路径：`(x1,x2,x3,...,xn)`

---
### 算法分类
#### 按掌握的信息分类
| 类型        | 核心思想               | 典型算法         |
| --------- | ------------------ | ------------ |
| 集中式路由选择算法 | 利用完整拓扑和链路开销计算路径    | [[LS 链路状态算法]] |
| 分散式路由选择算法 | 只依据局部信息，通过邻居交换逐步收敛 | [[DV 距离向量算法]] |
#### 按变化方式分类
| 类型   | 特点              |
| ---- | --------------- |
| 静态路由 | 手动配置，不自动适应网络变化  |
| 动态路由 | 自动学习路由，可随网络变化更新 |
#### 按链路开销是否反映负载分类
| 类型 | 特点 |
| --- | --- |
| 负载敏感算法 | 链路开销随拥塞程度变化 |
| 负载迟钝算法 | 链路开销不直接反映当前拥塞 |

> 当今路由选择算法多为 **负载迟钝**，因为拥塞状态很难稳定地转化为链路开销

### 分层次路由选择
> 因特网规模过大，不可能让所有路由器都掌握完整互联网拓扑

互联网先划分为若干 [[AS 自治系统]]，再分别处理：
- **AS 内部路由选择**：解决一个 AS 内部怎么转发
- **AS 间路由选择**：解决不同 AS 之间怎么互相到达

| 类型 | 范围 | 常见协议 | 说明 |
| --- | --- | --- | --- |
| IGP | AS 内部 | [[RIP 路由信息协议]]、[[OSPF开放最短路优先]] | 内部网关协议 |
| EGP | AS 之间 | [[BGP 边界网关协议]] | 外部网关协议 |

### 常见协议定位
| 项目       | [[RIP 路由信息协议]] | [[OSPF开放最短路优先]] | [[BGP 边界网关协议]] |
| -------- | -------------- | --------------- | ------------- |
| **封装协议** | UDP            | IP              | TCP           |
| **传播方式** | 逐跳             | 泛洪              | TCP 会话间传递     |
| **更新内容** | 全表，周期性         | 链路状态，事件驱动       | 路径属性，事件驱动     |
| **拓扑视图** | 无全局视图          | 拥有全图            | 无全图，仅路径属性     |
| **计算算法** | Bellman-Ford   | Dijkstra        | 策略驱动          |
| **收敛速度** | 慢              | 快               | 慢（但更稳定）       |
| **带宽占用** | 高（周期发全表）       | 中（仅更新变化）        | 低（TCP 控制精细）   |
| **扩展性**  | 差              | 较好              | 极强            |
| **应用场景** | 小型网络           | 企业内部网络          | 运营商/跨 AS 互联   |
