### 描述路由问题
> 在带权图中，为分组选择一条从源结点到目的结点的最低开销路径
![[Pasted image 20251022172535.png]]
- 网络表示为 `G=(N,E)`
- `N`：结点集合
- `E`：边集合
- `c(x,y)`：结点 `x` 与 `y` 之间边的开销
- 若 `x` 与 `y` 不直接相连，则 `c(x,y)=∞`
- 路径：`(x1,x2,x3,...,xn)`
### 算法分类
#### 按算法所得信息分类
| 类型        | 核心思想                            | 特点                 | 典型           |
| --------- | ------------------------------- | ------------------ | ------------ |
| 集中式路由选择算法 | 利用**完整的、全局的**网络信息计算最低开销路径       | 知道全网拓扑和链路开销        | [[链路状态算法LS]] |
| 分散式路由选择算法 | 各结点只依据**局部信息**，通过迭代和交换信息求最低开销路径 | 只知道与自己相连链路的开销，逐步收敛 | [[距离向量算法DV]] |
#### 按静态 / 动态分类
| 类型       | 特点                             |
| -------- | ------------------------------ |
| 静态路由选择算法 | 路由变化缓慢，通常由人工调整                 |
| 动态路由选择算法 | 路由随网络流量负载或拓扑变化而变化，易出现路由环路、路由振荡 |
#### 按负载敏感 / 负载迟钝分类
| 类型     | 特点                      |
| ------ | ----------------------- |
| 负载敏感算法 | 链路开销随拥塞程度变化，能反映当前链路负载情况 |
| 负载迟钝算法 | 不直接根据当前拥塞情况调整链路开销       |

> 当今路由选择算法多为**负载迟钝**，因为链路当前拥塞水平难以准确、稳定地反映到开销中
### 分层次路由选择
> 因特网规模过大，不可能让所有路由器都掌握整个互联网的完整拓扑
- 因特网采用**分层次路由选择**
- 先划分为若干 [[自治系统AS]]
- 自治系统内部采用一种自治系统内路由协议
- 自治系统之间采用自治系统间路由协议

| 类型         | 范围         | 常见协议                | 特点           |
| ---------- | ---------- | ------------------- | ------------ |
| 自治系统内部路由选择 | 一个 AS 内部   | [[开放最短路优先OSPF]]、RIP | 也称内部网关协议 IGP |
| 自治系统间路由选择  | AS 与 AS 之间 | [[边界网关协议BGP]]       | 也称外部网关协议 EGP |
