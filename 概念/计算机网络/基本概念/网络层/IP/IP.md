## IPv4
### IPv4数据报格式
- 最小首部 = 20B；最大首部：60B
![[Pasted image 20251019202309.png]]

| **字段名称**          | **功能与意义**         | **关键细节 / 数值备注**                                  |
| ----------------- | ----------------- | ------------------------------------------------ |
| **版本号**           | 标识 IP 协议版本        | 用于区分 **IPv4** 与 **IPv6**。                        |
| **首部长度**          | 确定 IP 数据报中数据的起始位置 | 因“选项”字段长度可变，故需此字段定位数据偏移。                         |
| **服务类型 (TOS)**    | 提供差分服务质量 (QoS)    | 让网络根据业务类型（如延迟敏感或吞吐量优先）处理报文。                      |
| **数据报长度**         | 报文的总长度（首部 + 数据）   | 16比特长，理论最大 **65535** 字节；实际受限于链路层 **MTU**。        |
| **标识 (ID)**       | 用于数据报分片与重组        | 标识同一原始数据报的所有分片。                                  |
| **标志 (Flags)**    | 控制分片行为            | **MF=0** 表示最后一片，**MF=1** 表示后续还有片（IPv6 不允许路由器分片）。 |
| **片偏移**           | 确定分片在原始报文中的位置     | 以 8 字节为偏移单位，指定该片在原始数据报中的相对位置。                    |
| **寿命 (TTL)**      | 防止数据报在网络中无限循环     | 经过每台路由器 **TTL 减 1**，减至 0 时报文被丢弃。                 |
| **协议 (Protocol)** | 指明上层传输层协议类型       | 常见值：**6 (TCP)**、**17 (UDP)**、**89 (OSPF)**。      |
| **首部校验和**         | 检测首部比特错误          | 采用**反码求和**算法；每经过一个路由器需重新计算（因 TTL 改变）。            |
| **源 IP 地址**       | 发送端的 IP 地址        | 标识数据报的起点。                                        |
| **目的 IP 地址**      | 接收端的 IP 地址        | 标识最终目的地，通常通过 **DNS** 查询获得。                       |
| **选项 (Options)**  | 扩展功能（可选）          | 仅 **IPv4** 支持，用于测试、调试或安全限制，长度可变。                 |
| **数据 (Payload)**  | 实际传输的有效载荷         | 包含 TCP/UDP 报文段或其他协议数据。                           |
> 由上述可见，一个IP数据报由总长为20字节的首部（不包括选项），这与[[TCP]]报文首段字节数相同（也为20）
### IPv4数据报分片
- IP数据报的长度受[[MTU]]限制
- 由于MTU受数据链路层协议影响，而不同链路由路由器连接，所以IPv4需要路由器做分片，而分成的部分自然称为片
- 为了让路由器提升性能，重组工作由端系统完成

| **字段名称**                  | **位数** | **作用**                                                                                                                                        |
| ------------------------- | ------ | --------------------------------------------------------------------------------------------------------------------------------------------- |
| **标识符 (Identification)**  | 16 bit | 用于标识属于同一个原始数据报的所有分片。**同一数据报的所有分片具有相同的标识符**，以便接收方重组。                                                                                           |
| **标志 (Flags)**            | 3 bit  | 控制和指示分片状态：<br>1. 保留位（始终为 0）<br>2. **DF** ：<br>若为 1，表示禁止分片<br>如果 MTU 不够，路由器会丢弃该包并返回 ICMP 错误<br>3. **MF** ：<br>若为 1，表示后面还有分片<br>若为 0，表示这是最后一个分片 |
| **片偏移 (Fragment Offset)** | 13 bit | 标识该分片在原始数据报中的相对位置<br>**注意：** 偏移量的单位是 **8 字节**<br>也就是说，实际偏移字节数 = $Offset \times 8$                                                             |
### IPv4编址
- 接口：主机与物理链路之间的边界
- 一个IP地址与一个接口相关联而非主机or路由器
#### IP地址
- 长度：32比特
- 点分十进制记法：每个字节（8bit）用其十进制形式书写，每个字节间用句点隔开（因此就是四个数）
#### 计算方式
| **概念名称**     | **定义与功能**                      | **计算方法 / 备注**                                     |
| ------------ | ------------------------------ | ------------------------------------------------- |
| **子网**       | 主机与路由器接口连接成的物理或逻辑网络。           | 也常被称为 IP 网络或直接称为“网络”。                             |
| **子网掩码 **    | 用于区分 IP 地址中网络位与主机位的位掩码。        | 连续的 1 代表网络位，连续的 0 代表主机位。                          |
| **网络号 **     | 代表整个子网的“门牌号”，不分配给具体主机。         | **计算：** $IP \text{ 地址} \text{ AND } \text{ 子网掩码}$ |
| **主机位 **     | 子网掩码中 0 对应的位，用于标识子网内的具体设备。     | 决定了该子网能容纳多少台主机。                                   |
| **可用主机地址范围** | 除去网络号和广播地址后，真正能分配给设备的地址。       | **范围：** `网络号 + 1` 至 `广播地址 - 1`。                   |
| **子网广播地址**   | 用于向该子网内所有主机发送数据的特殊地址。          | **计算：** 网络号中的主机位全置为 **1** (二进制)。                  |
| **有限广播地址**   | 全球通用的本地广播地址：`255.255.255.255`。 | 目的为本地网络内所有主机，路由器通常**不转发**。                        |
#### 网络号与主机号的作用
- 网络号用于标识一台主机**属于哪个网络**
- 主机号用于区分**同一网络里的不同设备**
- [[链路层寻址 & ARP|ARP]] 查的是**主机号**
## IPv6
- 32bit的IP地址空间将被用尽
### IPv6数据报结构
![[Pasted image 20251021145532.png]]
- 扩大的地址容量：IPv6的IP地址为128位，解决了IPv4的32位导致范围太小不够用的问题。引入了[[任播地址]]。
- 简化高效的40字节首部：舍弃了选项，首部成为定长
#### 具体内容
| **字段名称**   | **功能与意义**         | **关键细节 / 备注**                                                |
| ---------- | ----------------- | ------------------------------------------------------------ |
| **版本**     | 标识 IP 协议版本        | **值为 6**：表示 IPv6。_(注：虽然字段为 4 代表 IPv4，但在 IPv6 的语境下，该字段必须为 6)_ |
| **流量类型 **  | 区分不同类别的 IP 分组     | 类似于 IPv4 中的 **TOS**，用于支持服务质量 (QoS)。                          |
| **流标签**    | 标识属于同一业务流的分组      | 用于标识特定路径上的“流”（如实时视频流），提供更高优先级或特殊处理。                          |
| **有效载荷长度** | 首部之后的数据字节数        | **16比特长**。IPv6 采用固定首部（40字节），故不再需要“首部长度”字段。                   |
| **下一个首部**  | 标识紧跟在首部后的协议类型     | 指向运输层协议（如 TCP/UDP）或 IPv6 的**扩展首部**。                          |
| **跳限制**    | 限制分组在网络中的寿命       | 类似于 IPv4 的 **TTL**。每经一台路由器减 1，为 0 时分组被丢弃。                    |
| **源地址 **   | 发送端的 128 比特 IP 地址 | 极大扩展了地址空间，解决地址枯竭问题。                                          |
| **目的地址 **  | 接收端的 128 比特 IP 地址 | 最终接收方或路由选择路径上的下一个目标。                                         |
| **数据**     | 实际传输的有效载荷         | 包含上层协议数据，如 TCP 报文段、UDP 用户数据报等。                               |
#### 与IPv4的差异
- 不再有分片步骤，若数据报太大不能转发则丢掉数据报并向发送方返回一个“分组太大”的[[因特网控制报文协议ICMP|ICMP]]差错报文
- 不再有首部校验和，这是因为IP设计者觉得上层的检验操作够用了（？），并且每台路由器都要重新计算校验和太费时
- 选项：（可能）出现在IPv6的“下一个首部”指出的位置上
