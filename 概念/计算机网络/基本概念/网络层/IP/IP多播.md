### IP多播/组播
> 让源主机一次发送的单个分组可以抵达用一个组地址表示的若干目的主机
> 即发送时的IP：主机=1：n

---
### IP多播地址
| 地址类型 | 地址内容              |
| ---- | ----------------- |
| 源地址  | 源主机 IP            |
| 目的地址 | IP多播地址（IPv4的D类地址） |
多播地址范围：224.0.0.0 ~ 239.255.255.255
#### 协议字段值
UDP：17
IGMP：2
注意事项：
1. 不提供可靠交付（不使用TCP，仅适用UDP）
2. 只能用于目的地址
3. 不产生ICMP差错报文
#### 分类
1. 只在本局域网上进行硬件多播
2. 在互联网的范围内进行多播
### 硬件多播
> 将IP多播地址映射为对应的**多播 MAC 地址** 
#### 地址范围
`01-00-5E-00-00-00` ~ `01-00-5E-7F-FF-FF`
- 仅后23位用于多播
- 多播IP ：多播MAC = n : 1
### IGMP 网际组管理协议
> 使连接到本地局域网的**多播路由器**能获知本局域网上是否有主机加入、退出某多播组
#### 注意事项
1. 仅作用于本地子网
2. 既不知道某个多播组的总成员数量，也不了解这些成员分布在哪些网络
3. 封装在 IP 数据报中
4. IGMP被视为是整个IP协议的一个组成部分而非一个单独协议
#### 工作流程
1. 新加入主机向该多播组的多播路由器发送一个IGMP报文
	- 目的地址：多播IP地址
2. 路由器收到IGMP，利用多播路由选择协议将这种组成员关系通告给其他多播路由器
3. 多播路由器周期性查询本地局域网上的主机，以探测各多播组是否仍有活跃成员
	- 只要有一个主机回应就认为本网络仍处于活跃状态
	- 若多次查询后仍无主机回应则认为该网络已无成员，停止广播该网络组成员关系
