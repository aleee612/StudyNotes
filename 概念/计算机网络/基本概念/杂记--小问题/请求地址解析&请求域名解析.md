# 请求地址解析是什么？
> 浏览器把你输入的 URL 拆开、理解它的结构。
- 也叫 **URL 解析 / 请求解析**。
- 比如你输入：
```
https://www.example.com:8080/path/index.html?x=1#top
```
- 浏览器会把它分成：

|部分|意义|
|---|---|
|`https`|协议|
|`www.example.com`|主机名|
|`8080`|端口号|
|`/path/index.html`|路径|
|`?x=1`|查询参数|
|`#top`|锚点（不会发给服务器）|

- 这个过程完全发生在浏览器内部，不需要网络不涉及 DNS
- 它只是帮你把 URL 按照 RFC 标准拆开，知道应该连接谁、请求什么文件。
---
# 域名解析是什么？
> **把域名（[www.example.com）变成](http://www.example.xn--com\)-ed1g666e/) IP 地址（比如 93.184.216.34）。**
- 这是 **DNS 做的事情**。
- 浏览器拿到主机名后，会走 DNS 查询流程：
	- 先查本地缓存
	- 再查系统缓存
	- 再查 DNS 服务器
	- 逐级向根域名服务器、顶级域名服务器查询
	- 最终得到 IP 地址
- 这个过程一定涉及网络通信（除非缓存命中）
- 用来解决的关键问题是：
> 服务器到底在哪里？它的 IP 是多少？
---
# 核心区别
- 请求地址解析：解析 URL 的结构。谁做？浏览器。是否联网？不需要。
- 域名解析：把域名变成 IP 地址。谁做？DNS 系统。是否联网？需要（除非缓存）。
---
# 对比

|项目|请求地址解析|域名解析|
|---|---|---|
|谁做|浏览器 / 客户端|DNS 系统（递归 DNS 等）|
|是否联网|❌ 不需要|✔ 需要（除缓存）|
|解析内容|URL 内部结构|域名 → IP|
|作用|知道要访问什么资源|知道服务器在哪里|
|是否涉及协议|主要是浏览器逻辑|DNS 协议（[[UDP]] 或 [[TCP]]）|
|示例|拆出协议、路径、参数|example.com → 93.184.x.x|
