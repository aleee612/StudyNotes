- 当TCP发送一个大文件时，通常是将文件分为长度是[[MSS最大报文长度|MSS]]的若干块
### 首部
- [[序号和确认号|32bit序号+32bit确认号]] 
	=> 实现[[可靠数据传输原理|可靠数据传输服务]] 
- 16bit **接收窗口** 
	=> 用于流量控制（指示接收方愿意接收的字节数）
- 4bit **首部长度** 
	=> 首部长度 = 字段值 × 4 字节（填充字段保证首部为4bit的倍数）
- **选项**字段 
	=> 协商[[MSS最大报文长度|MSS]] 、高速网络环境用作 窗口调节因子
	最长可达40B
- 6bit **标志字段** 
- 2B 检验和
	1. 检查范围：首部+数据
	2. 伪首部协议字段：将 17 改为 6
	3. 伪首部长度字段：TCP长度
- 2B 紧急指针
	1. 仅在 URG=1 时有效
	2. 指出紧急数据字节数（位于数据开头）
	3. 即使接收窗口=0，仍可发送紧急数据
- 最小首部： 20B；最大首部：60B
### 6bit标志字段
| TCP 标志位 | 含义     | 说明                                                              |
| ------- | ------ | --------------------------------------------------------------- |
| **ACK** | 确认应答   | ACK=1 时表示确认号有效；确认号表示**期望收到对方下一个字节的序号**，也就是说明 `x` 之前（含 x-1）都收到了。 |
| **SYN** | 同步序号   | 用于建立连接（三次握手中的前两步）。                                              |
| **FIN** | 结束发送   | 用于正常关闭连接（四次挥手）。                                                 |
| **RST** | 重置连接   | **立即终止连接**或拒绝无效连接，不走四次挥手，粗暴版断线。<br>或表示必须**释放**连接，然后**重新建立**连接   |
| **CWR** | 拥塞窗口减少 | 用于明确拥塞提醒。发送方告诉对方 “我已经降低窗口了”。                                    |
| **ECE** | 显式拥塞通知 | 代表网络中出现拥塞，用于 ECN 机制下的拥塞报告。                                      |
| **PSH** | 推送数据   | 置位后要求接收方**立刻将数据交给上层应用**，不必在缓冲区中等待积累。                            |
| **URG** | 紧急数据   | 说明报文中有被上层标记为“紧急”的数据，需要优先处理。                                     |
