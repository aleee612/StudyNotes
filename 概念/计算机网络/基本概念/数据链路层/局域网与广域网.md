### 广域网
| 特性   | 位填充              | 字符填充           |
| ---- | ---------------- | -------------- |
| 操作单位 | bit              | byte           |
| 典型协议 | HDLC，PPP（同步传输）   | PPP（异步传输）      |
| 冲突来源 | 比特模式（如 01111110） | 特殊字符（FLAG/ESC） |
| 方法   | 插 0              | 插 ESC          |
| 复杂度  | 更底层              | 更接近应用层         |
| 可扩展性 | 强（与编码无关）         | 依赖字符集          |
> [!note]- 透明传输 = 在数据流中打断可能引起误解的模式
> **位填充：** 如帧定界符`01111110`
> 数据里如果也出现连续 5 个 1，会和定界符“前半段”冲突
> **字符填充：** 帧结构FLAG + DATA + FLAG， FLAG = `0x7E`
> 如果 DATA 中也出现 `0x7E`，接收方会误认为帧结束了
#### HDLC（不在大纲）
 - 面向**比特** 的 **数据链路层协议**
 - 用于在点对点或点对多点通信中提供可靠的数据传输
#### PPP 协议

- 用于在两个**点对点**连接的网络之间传输数据
- 最初为拨号连接设计，后来被广泛用于建立各种类型的点对点连接
- 包括 DSL（数字用户线路）、ISDN（综合业务数字网）和串口连接等
##### PPP组成部分
**LCP：**
- 作用：建立，配置，测试数据链路链接，认证
**NCP：**
- 作用：配置网络层协议
**封装IP数据报：**
- 长度受MTU限制
