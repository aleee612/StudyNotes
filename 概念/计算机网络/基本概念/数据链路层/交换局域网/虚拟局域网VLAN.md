解决问题：
- 缺乏流量隔离：
	- 限制部分其他局域网的广播流量（如ARP和DHCP报文）将改善性能
- 交换机的无效使用：
	- 组太多传统的链路层交换机有资源浪费
- 管理用户：
	- 不同组之间难以通信
### VLAN
- 在一个VLAN中的主机**彼此通信**
- 在一个基于端口的VLAN中，交换机接口将被**分组**，每一个组构成一个VLAN
- 每个VLAN中的端口形成一个**广播域**，其中的接口可以**互相通信** 
### VLAN干线链接trunk
- 干线端口：trunk端口
- 所有的VLAN都能通过干线端口传输
### 802.1Q：跨越VLAN干线的帧
**作用：**
- 通过该帧，判定取得的帧是哪个VLAN的

| **字段名称**           | **长度**     | **作用说明**                                             |
| ------------------ | ---------- | ---------------------------------------------------- |
| 标签协议标识             | 16 bit     | 固定值为 `0x8100`，用于供交换机识别为802.1Q帧                       |
| 优先级/PCP            | 3 bit      | 用于 服务质量<br>取值 0-7，值越高优先级越高<br>当网络拥塞时，交换机会优先转发高优先级的数据 |
| **CFI/DEI** (丢弃指示) | 1 bit      | 主要用于兼容令牌环网<br>或在拥塞时标记哪些帧可以被优先丢弃                      |
| **VLAN ID (VID)**  | **12 bit** | **最核心的部分**。用于标识该帧属于哪个 VLAN。取值范围是 **0 ~ 4095**。       |
**标签的发送与接收：** 
- VLAN发送侧交换机 + VLAN标签 -> VLAN干线接受侧解析并删除
