### 物理媒介
| 参数    | 10BASE-5 | 10BASE-2 | 10BASE-T | 10BASE-FL |
| ----- | -------- | -------- | -------- | --------- |
| 传输媒体  | 粗缆       | 细缆       | 双绞线      | 光纤对       |
| 编码    | 曼彻斯特编码   | 曼彻斯特编码   | 曼彻斯特编码   | 曼彻斯特编码    |
| 拓扑结构  | 总线形      | 总线形      | 星形       | 点对点       |
| 网络适配器 | 早期以太网    | 早期以太网    | 现代以太网    | 现代以太网     |
| 双工模式  | 仅半双工     | 仅半双工     | 半双工/全双工  | 半双工/全双工   |
- `XXX-数字` or `数字 + G`：该标准的速率，单位为Mbps
- BASE：[[基带以太网]] 
- T：双绞铜线
- FL：光纤
> [!warning] 做题须知
> 10BaseT 是 10 Mbps 以太网标准，属于经典以太网。
一、在经典以太网里：
1). 最小帧长 = 64 字节
2). 发送 1 bit 在 10 Mbps 下需要：
$$1/10 000 000=0.1μs$$
二、计算最小帧发送时间：
64 字节 = 512 bit
$$512×0.1μs=51.2μs$$

### 设备
#### 集线器
| **特性**   | **详细说明**                               |
| -------- | -------------------------------------- |
| **工作层次** | 物理层                                    |
| **拓扑结构** | 星形                                     |
| **冲突域**  | 整个集线器属于一个**冲突域**<br>同一时刻只能有一个设备发送数据    |
| **广播域**  | 整个集线器属于一个**广播域**                       |
| **通信模式** | **半双工**<br>设备不能同时发送和接收数据               |
| **安全性**  | 较低<br>因为数据是广播的，网络中的任何设备理论上都能抓取到发往他人的数据 |
**作用：**
- 将从接口接收到的比特向**所有**其他接口转发
- 中继：
	接收到信号后对其进行**整形**和**放大**
- 延长距离：
	通过转发有效克服电缆长度的限制，**扩大**网络的**覆盖范**围
**碰撞与解决：**
- 当某集线器同时从两个不同的接口接受到帧，将出现碰撞
- 生成该帧的节点必须重新传输该数据
#### 交换机
- 优于集线器，是无碰撞的，而且是名副其实的转发分组
- [[概念/计算机网络/基本概念/数据链路层/交换局域网/链路层交换机]] 
#### 网桥与交换机的区别
| 方面       | 网桥                   | 交换机                        |
| -------- | -------------------- | -------------------------- |
| **端口数量** | 通常较少（2～4 个）          | 通常很多（几十个甚至上百个）             |
| **性能**   | **软件**转发，处理能力较弱      | **硬件**转发（ASIC 芯片），转发速度更快   |
| **功能**   | **简单**地转发帧，适合小型或实验网络 | 支持 **VLAN**、端口镜像、链路聚合等高级功能 |
| **使用场景** | 用于连接两个小型网络           | 用于构建现代企业内部网络（LAN）          |

### 以太网帧结构
#### 传统帧结构（有线）
| 字段       | 作用                                                                              |
| -------- | ------------------------------------------------------------------------------- |
| **前同步码** | 前七个字节为10101010重复，最后一个字节为10101011<br>**前七位作用：**<br>	**唤醒**接受适配器，并将其时钟与发送方的**同步** |
| **目的地址** | 目的适配器的**MAC地址**<br>当收到为 广播地址 或者 本地MAC 则上交至网络层                                   |
| **源地址**  | 发送方的适配器的MAC地址                                                                   |
| **类型字段** | 允许以太网**复用**多种**网络层协议**，当字段值为0806（16进制）为ARP<br>**作用：** 与上一层的协议结合                 |
| **数据字段** | IP数据报内容                                                                         |
| **CRC**  | [[差错检测和纠正#循环冗余检测CRC\|循环冗余检测]] -> FCS                                            |
