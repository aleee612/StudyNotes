> **summary：**
> DHCP获取IP 
> → ARP获取网关MAC 
> → DNS解析域名 
> → TCP三次握手 
> → TLS握手 
> → HTTP请求（携带Cookie）
> → 服务器响应 → 浏览器渲染页面

### 一、接入网络
#### 1. DHCP 获取网络配置
PC 刚开机：交换机广播
``` title=交换机广播
DHCP Discover
→ UDP
→ IP(0.0.0.0 → 255.255.255.255)
→ Ethernet(MAC → FF:FF:FF:FF:FF:FF)
```
DHCP服务器回复：
```title=DHCP服务器回复
Offer
Request
ACK
```
最终获得：
- IP地址
- 子网掩码
- 默认网关
- DNS服务器地址
---
#### 2. ARP 获取网关MAC
已知：
```
网关IP
```
未知：
```
网关MAC
```
发送ARP Request：
```
Who has 网关IP?
Tell PC_IP
```
广播发送

网关收到后回复：
```
ARP Reply
网关IP -> 网关MAC
```
PC缓存ARP表。

得到：
```
网关IP ↔ 网关MAC
```
---
### 二、访问网站
假设访问：
```
www.baidu.com
```
#### 1. DNS解析
DNS查询：
```
www.baidu.com ?
```
封装：
```title=DNS封装
DNS
→ UDP
→ IP(PC → DNS)
→ Ethernet(PC → 网关)
```

注意：
这里链路层目的MAC始终是：
```
下一跳MAC
```
不是DNS服务器MAC。

DNS返回：
```
www.baidu.com
↓
110.xxx.xxx.xxx
```

---
#### 2. TCP三次握手
得到服务器IP后：
```
SYN
SYN+ACK
ACK
```
建立TCP连接

封装：
```title=TCP封装
TCP
→ IP(PC → 服务器)
→ Ethernet(PC → 网关)
```
经过多个路由器转发
这里不需要写 BGP
408里只需写：
```
经过多个路由器转发
```
BGP属于自治系统间路由协议，不是每次上网都要专门提

---
### 三、HTTPS
如果是HTTPS：
TCP建立后先进行：
#### TLS握手
客户端：
```
Client Hello
```
发送：
- TLS版本
- 支持的加密套件
- 随机数
服务器：
```
Server Hello
```
发送：
- 证书
- 公钥
- 选择的加密套件
客户端：
```
验证CA签名
```
验证证书合法性

随后：
```
协商会话密钥
```
建立对称加密信道

---
### 四、HTTP通信
TLS完成后：
```
HTTP Request
```
例如：
```title=http
GET /index.html
Host: www.baidu.com
Cookie: xxx
```
发送给服务器。

---
#### Cookie传输
Cookie本质是HTTP首部：
服务器：
```http
Set-Cookie: uid=123
```
浏览器保存

后续请求：
```http
Cookie: uid=123
```
自动携带

---
### 五、服务器响应
服务器返回：
```http
HTTP/1.1 200 OK
```
携带：
- HTML
- CSS
- JS
- 图片资源
浏览器解析渲染页面

私网IP

路由器
