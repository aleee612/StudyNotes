### HTTP 常用方法

| 方法      | 作用          | 是否幂等 | 是否安全 |
| ------- | ----------- | ---- | ---- |
| GET     | 获取资源        | √    | √    |
| POST    | 提交数据、创建资源   | ×    | ×    |
| HEAD    | 获取资源首部信息    | √    | √    |
| PUT     | 更新或创建资源     | √    | ×    |
| DELETE  | 删除资源        | √    | ×    |
| OPTIONS | 查询服务器支持的方法  | √    | √    |
| TRACE   | 回显请求，用于测试   | √    | √    |
| CONNECT | 建立隧道（HTTPS） | ×    | ×    |

---
### 各方法详细说明
#### GET
##### 功能
请求获取指定资源
##### 特点
- 最常用
- 参数通常放在 URL 中
- 不应修改服务器资源
- 可缓存
- 可被书签保存
##### 例：
```http
GET /index.html HTTP/1.1
Host: www.example.com
```
服务器返回：
```http
HTTP/1.1 200 OK
```

---
#### POST
##### 功能
向服务器提交数据
##### 特点
- 常用于提交表单
- 数据放在实体部分（Body）
- 可能导致资源变化
- 不可缓存
##### 例：
```http
POST /login HTTP/1.1

username=Tom&password=123
```
##### 应用
- 登录
- 上传文件
- 提交评论

---
#### HEAD
##### 功能
只获取响应首部，不返回实体
##### 返回：
```
状态行
首部字段
（无实体）
```
##### 用途
① 判断资源是否存在
② 获取：
- Content-Length
- Last-Modified
- Content-Type
③ 检查文件大小
④ 判断缓存是否失效
##### 特点
本质上：
```
HEAD = GET − 消息体
```

---
#### PUT
##### 功能
上传或更新指定资源
```http
PUT /student/1001
```
若不存在 - 创建
若存在 - 覆盖原资源
##### 特点
重复执行结果相同：
```
PUT 是幂等的
```

---
#### DELETE
##### 功能
删除资源
```http
DELETE /student/1001
```
DELETE 是幂等的

---
#### OPTIONS
##### 功能
询问服务器支持哪些方法
服务器可能返回：
```http
Allow: GET,POST,HEAD
```
##### 应用
跨域请求（CORS）预检：
```http
OPTIONS /api
```

---
#### TRACE
##### 功能
服务器将收到的请求原样返回
用于：
- 调试
- 路径测试
返回：
```http
TRACE / HTTP/1.1
Host: xxx
```
服务器：
```http
TRACE / HTTP/1.1
Host: xxx
```
现在多数服务器默认关闭

---
#### CONNECT
##### 功能
建立 TCP 隧道
主要用于：
##### HTTPS
过程：
```
浏览器
 ↓ CONNECT
代理服务器
 ↓
目标服务器
```
建立隧道后：
```
TLS 加密数据直接通过
```

---
### 安全方法
> 不会对服务器资源造成修改
> GET、HEAD、OPTIONS、TRACE

---
### 幂等方法
> 执行一次和执行多次效果相同
> GET、HEAD、PUT、DELETE、OPTIONS、TRACE

---
### GET 与 POST 的区别（重点）

|比较|GET|POST|
|---|---|---|
|功能|获取资源|提交数据|
|参数位置|URL|Body|
|长度限制|URL长度限制|理论无限|
|是否缓存|可缓存|一般不缓存|
|是否书签|可以|不可以|
|是否幂等|√|×|
|是否安全|√|×|
##### 注意
GET 一定不能携带消息体吗？
答案：
**不是。**
HTTP 协议没有禁止 GET 带 Body
只是：
- 实际应用几乎不用；
- 很多服务器会忽略。
