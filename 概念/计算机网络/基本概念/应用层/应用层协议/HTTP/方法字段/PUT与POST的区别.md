## PUT：幂等、替换式更新
PUT 的语义是：
> **客户端给你一个资源的完整新版本，请用它替换旧的。**
- 特点：
- **幂等**：重复执行多次，结果一样  
    （你发 100 次同样的 PUT，最终资源还是同一个内容）
- **资源位置由客户端决定**  
    客户端告诉服务器：  
    `PUT /user/123` → “我就在这里更新 123”
- 常用于 **更新 或 创建已知 URI 的资源**
例子：
```http
PUT /user/123
Content-Type: application/json

{
  "name": "Alice",
  "age": 20
}
```
意思是：  
把 `/user/123` 这个资源替换成这个 JSON 内容。

## POST：非幂等、创建或提交动作
POST 的语义是：
> **发送数据让服务器处理，处理方式由服务器决定。**
- 特点：
	- 不是幂等的  
	    发两次等于创建两个订单、两条记录……
	- 资源位置由服务器决定
	- 常用于：
	    - 创建新资源（如注册、下单）
	    - 执行某种动作（登录、提交表单）
	    - 不适合用明确 URI 表达的操作
例子：
```http
POST /users
Content-Type: application/json

{
  "name": "Alice",
  "age": 20
}
```
服务器可能返回：
```
201 Created
Location: /users/567
```
资源 ID 由服务器生成 → 非幂等。
