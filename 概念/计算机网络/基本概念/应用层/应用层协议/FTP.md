### FTP
#### 基础架构与特性
- **运输层协议**：**TCP**
-  **带外传送**：由于有**分离的控制连接**，所以称FTP控制信息为带外传送
- **连接特性**：所以FTP是**有连接的**
- **扩展**：**NFS**
#### 控制连接 vs 数据连接
| **连接类型** | **作用**            | **服务器监听端口**                | **生命周期与特性**            |
| -------- | ----------------- | -------------------------- | ---------------------- |
| **控制连接** | 传输控制信息，如连接请求，传送请求 | **21**                     | **长期保持**，不会因为文件传输结束而关闭 |
| **数据连接** | 实际的数据传送（文件）       | 主动模式为 **20** / 被动模式为**随机** | **临时**，传完文件就断          |
#### 主动模式 (PORT) vs 被动模式 (PASV)
| **模式**                    | **发起方** | **服务器监听端口** | **详细连接流程**                                                              |
| ------------------------- | ------- | ----------- | ----------------------------------------------------------------------- |
| **主动模式 PORT**<br>_(默认模式)_ | **服务器** | **20**      | 1. 客户端随机开放端口，发送 PORT 命令告知服务器<br>2. **服务器接收并通过 20 端口与客户端口连接**            |
| **被动模式 PASV**             | **客户端** | **随机**      | 1. 客户端随机开放端口，发送 PASV 命令告知服务器<br>2. 服务器接收并开放随机端口，告知客户<br>3. 客户发起连接至服务器端口 |
#### 安全性分析与优化
- **不安全缺陷**：在传输过程中使用**明文传输**用户名、密码和数据
- **优化方案**：
    - **SFTP**：基于 **SSH** 的文件传输协议，加密传输数据
    - **FTPS**：基于 **SSL/TLS** 的 FTP，加密传输数据
#### 请求用户与权限划分
| **用户类型**                 | **登录凭证 / 用户名**                                  | **权限和操作范围**                                                                      | **权限级别**   |
| ------------------------ | ----------------------------------------------- | -------------------------------------------------------------------------------- | ---------- |
| **Real <br>(真实用户)**      | 独立登录权限                                          | 1. 可以访问自己的主目录<br>2. 可以执行各种文件操作（如上传、下载、删除等）                                       | **最高权限**   |
| **Guest <br>(访客用户)**     | 权限受限用户                                          | 1. 只能访问自己的主目录，并且可能无法执行某些文件操作（如删除或重命名）<br>2. 权限和操作范围由FTP服务器管理员进行配置和限制，以增加服务器的安全性。 | **权限受限**   |
| **Anonymous <br>(匿名用户)** | 使用 “anonymous” 作为用户名，并提供一个**有效的电子邮件地址**作为密码进行登录 | 1. 未注册但允许访问公开资源<br>2. 可以访问服务器上的公共目录和文件<br>3. 但权限和操作范围受到限制                        | **受限公共访问** |

### FTP 的文件传输模式（文本 vs 二进制）
- **ASCII 模式（文本模式）**：
    - 用于传输纯文本文件（ `.txt`, `.html`, `.xml`）
    - 它会自动转换不同操作系统之间的 **换行符**（ Windows 的 `\r\n` 与 Linux 的 `\n`）
    - 如果用它传输压缩包或图片，会导致文件损坏
- **Binary 模式（二进制模式/图像模式）**：
    - 不对数据进行任何转换，逐字字符原样复制传输
    - 用于传输程序、压缩包（`.zip`, `.tar`）、图片、视频等非纯文本文件
    - **在无法确定文件类型时，闪存默认推荐使用二进制模式**

### FTP 典型状态码（1位数字含义与常见码）
- **首位数字的核心含义（必背）**：
    - **`1xx`**：提示信息（动作已开始，正在进行）
    - **`2xx`**：**成功**（动作已顺利完成）
    - **`3xx`**：**正文重定向/需要进一步信息**（如需要输入密码）
    - **`4xx`**：**瞬时失败**（暂时性错误，客户端可以稍后重试）
    - **`5xx`**：**永久失败**（命令无法执行，如权限不足、文件不存在）
- **常见经典状态码**：
    - `220`：服务就绪（连上服务器后看到的第一条信息）
    - `331`：用户名正确，需要输入密码（`User name okay, need password`）
    - `230`：用户登录成功
    - `550`：文件未找到或没有访问权限

### 有状态协议的特性
> FTP 是一个有状态的协议
- **含义**：服务器在整个会话期间，必须记住用户的登录状态、当前工作目录
- **对比**：
	- HTTP 每次请求都是独立的
	- FTP 必须在“有状态”的控制连接下才能正常工作

### 常用 FTP 交互命令
| **原始命令**                | **对应操作/含义**                               |
| ----------------------- | ----------------------------------------- |
| **`USER`** / **`PASS`** | 输入用户名 / 输入密码                              |
| **`LIST`**              | 列出当前目录下的文件列表（相当于 Linux 的 `ls`）            |
| **`CWD`**               | 切换工作目录（Change Working Directory，相当于 `cd`） |
| **`RETR`**              | 下载文件（Retrieve，从服务器获取文件）                   |
| **`STOR`**              | 上传文件（Store，向服务器写入文件）                      |
| **`QUIT`**              | 退出并断开控制连接                                 |

### NFS 与 FTP 的深层本质区别
- **FTP（文件传输）**：
  必须把文件**完整地下载**到本地客户端，才能在本地进行修改或读取
- **NFS（网络文件系统/文件共享）**：
  1. 允许客户端**直接透明地挂载**远程目录
  2. 客户端修改文件时，是通过网络直接对服务器上的数据进行读写，**不需要**先下载整个文件
  3. 常用于 Linux 局域网集群
