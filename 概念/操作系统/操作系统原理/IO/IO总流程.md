#### 1. 用户请求阶段（用户态）
- **动作：** 用户程序执行一条 I/O 指令 %% 如 C 语言的 scanf 或 Python 的 input %%
- **核心点：** 用户程序不能直接操作硬件，它会发起一个系统调用 ，将控制权交给操作系统。此时，进程通常会进入阻塞状态
#### 2. 操作系统处理阶段（内核态）
- **设备无关性软件：** 检查参数是否**合法**，查看请求的数据是否已经在缓存中
- **设备驱动程序：** 
	- 如果缓存没有，OS 调用对应设备的驱动程序
	- 驱动程序负责将抽象的请求转换为硬件能懂的**命令序列**
- **下达指令：** CPU 向 I/O 控制器（如磁盘控制器）的寄存器写入命令（读/写）、地址和数据
#### 3. 硬件执行阶段（物理层）
- **忙碌等待/DMA：** 
	- 设备控制器指挥硬件（如磁盘磁头）开始读写
    - 如果是 **DMA (直接存储器存取)** 模式，硬件会直接把数据从设备搬运到内存中，不需要 CPU 参与
- **完成任务：** 硬件操作完成后，设备控制器向 CPU 发出一个**中断信号**
#### 4. 中断处理阶段（核心转折点）
- **硬件响应（中断隐指令）：** 
	- CPU 暂停当前任务，保存断点（PC/PSW）
	- 根据中断向量表找到**中断服务程序 (ISR)** 的入口
- **软件处理（ISR）：** 
	1. 保存现场。
	2. **获取设备输入：** 从控制器寄存器或内存缓冲区读取数据
	3. 恢复现场
#### 5. 返回用户程序阶段
- **唤醒进程：** 操作系统发现数据已就绪，将之前阻塞的进程状态改为“就绪态”
- **数据传递：** 内核将数据从内核缓冲区拷贝到用户程序的内存空间
- **继续执行：** 用户程序拿到数据，接着运行下一条指令
