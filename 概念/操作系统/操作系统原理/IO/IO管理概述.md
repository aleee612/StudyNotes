### IO设备
#### 分类 
| **分类维度**   | **设备类型** | **核心特征**                     | **典型示例**    |
| ---------- | -------- | ---------------------------- | ----------- |
| **信息交换单位** | **块设备**  | 以数据块为单位，速率高，可寻址（随机访问）        | 磁盘          |
|            | **字符设备** | 以字符为单位，速率低，不可寻址，常采用中断异步通信    | 打印机、交互式终端   |
| **传输速率**   | **低速设备** | 每秒几个字节至数百字节                  | 键盘、鼠标       |
|            | **中速设备** | 每秒数千字节至数十千字节                 | 激光打印机       |
|            | **高速设备** | 每秒数百千字节至数百兆字节                | 磁盘          |
| **使用特性**   | **人机交互** | 用于用户与计算机交互                   | 键盘、显示器、打印机  |
|            | **存储设备** | 用于持久化存储数据                    | 磁盘、磁带、光盘    |
|            | **网络通信** | 用于计算机间通信                     | 网卡、调制解调器    |
| **共享属性**   | **独占设备** | 一段时间内只允许一个进程使用               | 打印机（低速设备居多） |
|            | **共享设备** | 允许一段时间内多个进程逻辑上并发访问（分时交替）     | 磁盘          |
|            | **虚拟设备** | 通过 SPOOLing 技术将独占设备改造为逻辑共享设备 | 利用磁盘作为缓冲区实现 |
#### 接口（设备控制器）
| **组成部分**      | **连接对象**    | **功能说明**                    | **包含信号线**               |
| ------------- | ----------- | --------------------------- | ----------------------- |
| **与 CPU 的接口** | 连接 CPU 和控制器 | 实现 CPU 与控制器间的通信             | **数据线**、**地址线**、**控制线** |
| **与设备的接口**    | 连接控制器与设备    | 一个控制器可连接一个或多个设备             | 数据信号、控制信号、状态信号          |
| **I/O 逻辑**    | 核心控制单元      | 对 CPU 的命令进行译码，根据地址识别并控制指定设备 | -                       |

| **分类维度**     | **接口类型**   | **特点 / 定义**                   |
| ------------ | ---------- | ----------------------------- |
| **数据传送方式**   | **并行接口**   | 一个字节或字的所有位**同时**传输，速度快但适合短距离。 |
|              | **串行接口**   | 一位一位地**顺序**传输，接口需负责“并/串”转换。   |
| **主机访问控制方式** | **程序查询接口** | CPU 需不断轮询检查 I/O 状态，效率较低。      |
|              | **中断接口**   | I/O 设备完成后主动“通知” CPU，效率显著提高。   |
|              | **DMA 接口** | 直接存储器存取，数据传输不经过 CPU，适合高速大量数据。 |
| **功能灵活性**    | **可编程接口**  | 可以通过软件（编程）来改变接口的工作模式或功能。      |
|              | **不可编程接口** | 功能由硬件电路固定，无法通过软件修改。           |
| **设备类型**     | **字符设备接口** | 连接以字符为单位的设备（如键盘、打印机）。         |
|              | **块设备接口**  | 连接以数据块为单位的设备（如硬盘、光驱）。         |
|              | **网络设备接口** | 连接网络通信相关的设备（如网卡）。             |
#### 端口（本质寄存器）
**数据寄存器：** 
- 缓存从设备送来的输入数据，或暂存CPU发出的输出数据
**状态寄存器：** 
- 保存设备的状态或执行结果，供CPU读取
**控制寄存器：** 
- 由CPU写入，用于启动操作或设置设备的工作模式
##### 常见编址方式
| **特性**   | **独立编址 (I/O 映射方式)**                                 | **统一编址 (内存映射方式)**                                     |
| -------- | --------------------------------------------------- | ----------------------------------------------------- |
| **地址空间** | I/O 端口与内存地址空间**逻辑上完全分离**，可以有重合的地址值。                 | I/O 端口与内存**共用同一个地址空间**，占用部分主存地址。                      |
| **访问方式** | 必须使用**专用 I/O 指令**（如 x86 的 `IN` / `OUT`）。            | 使用**普通的访存指令**（如加载 `LD` / 存储 `ST`）。                    |
| **优点**   | 1. 译码简单，寻址快<br>2. 专用指令使程序易读，便于调试                    | 1. 编程灵活，无需专用指令<br>2. 端口可获得较大的编址空间<br>3. 可利用虚拟存储管理实现保护 |
| **缺点**   | 1. 指令功能有限，灵活度低<br>2. CPU 需提供两组控制信号（读/写 I/O 和 读/写内存） | 1. **占用主存空间**，减少了可用内存<br>2. 需根据完整地址判断，译码较慢且电路复杂       |
### IO控制方式
| **特性**       | **程序查询方式**                 | **中断驱动方式**                   | **DMA 方式**                                  | **I/O 通道方式**             |
| ------------ | -------------------------- | ---------------------------- | ------------------------------------------- | ------------------------ |
| **数据传输单位**   | **字/字节**                   | **字/字节**                     | **数据块**                                     | **一组数据块**                |
| **数据流向**     | 设备 → CPU → 内存              | 设备 → CPU → 内存                | **设备 → 内存** (直连)                            | **设备 → 内存** (自主)         |
| **CPU 干预频率** | **极高**：CPU 全程忙等，直到 I/O 完成。 | **高**：每个字传输完成后都要中断 CPU。      | **中**：仅在块的开始和结束时干预。                         | **极低**：仅在整个任务开始和结束时干预。   |
| **硬件成本**     | 低（几乎不需额外硬件）                | 中（需中断控制器）                    | 较高（需 DMA 控制器）                               | 高（需专门的通道硬件）              |
| **并行性**      | **无**：CPU 与 I/O 串行工作。      | **部分并行**：CPU 与 I/O 设备在处理时并行。 | **高度并行**：数据传输不占用 CPU 资源。                    | **最高**：多设备、多子系统全并行。      |
| **优点**       | 控制简单，硬件开销小。                | 提高了 CPU 利用率，无需忙等。            | 减少了 CPU 对批量数据的干预（仅在数据块传送开始和结束时需要CPU），传输速度快。 | 彻底解放 CPU，支持复杂的 I/O 任务管理。 |
| **缺点**       | **CPU 利用率极低**，资源浪费严重。      | 频繁中断会产生巨大的**上下文切换开销**。       | 逻辑复杂；每个设备通常需要独占 DMA 资源。                     | 硬件极其复杂，通常只用于大型主机。        |
#### DMA方式中的四类寄存器
| **寄存器名称**    | **缩写**  | **核心功能**                                  | **在不同数据流向中的作用**                                     |
| ------------ | ------- | ----------------------------------------- | --------------------------------------------------- |
| **命令/状态寄存器** | **CR**  | **控制与反馈**：暂存 CPU 发来的 I/O 命令，或记录设备的当前运行状态。 | 作为 CPU 与 DMA 控制器沟通的“信箱”。                            |
| **内存地址寄存器**  | **MAR** | **定位内存**：存放数据在内存中的起始物理地址。                 | **输入时**：指向写入目标地址。<br><br>  <br><br>**输出时**：指向读取源地址。 |
| **数据寄存器**    | **DR**  | **数据中转**：暂存单次传输的数据单元（如一个字）。               | 起到“搬运站”的作用，负责设备与内存间的字转换。                            |
| **数据计数器**    | **DC**  | **传输计数**：记录本次任务还需传送的字节/字总数。               | 每传输完一个单位，DC 减 1，减至 0 时发出中断请求。                       |

### 层次结构
| **名称**         | **主要功能**                                               | **典型实例**                         |
| -------------- | ------------------------------------------------------ | -------------------------------- |
| **用户层 I/O 软件** | 提供用户交互接口；<br>发出 I/O 请求；<br>进行格式化输出                     | `printf`、`scanf`、假脱机系统（SPOOLing） |
| **设备独立性软件**    | 映射逻辑设备到物理设备；<br>检查用户权限；<br>分配与回收设备；<br>**提供统一的系统调用接口** | 逻辑设备名映射、缓冲区管理（单缓冲/双缓冲）           |
| **设备驱动程序**     | 将抽象的 I/O 请求转换为具体的硬件指令；<br>控制设备控制器工作                    | 显卡驱动、声卡驱动、磁盘驱动程序                 |
| **中断处理程序**     | 处理 I/O 完成后的中断信号；<br>进行进程上下文切换；<br>将数据搬运结果告知驱动          | I/O 完成后的中断服务例程 (ISR)             |
| **硬件/设备控制器**   | 物理层执行 I/O 操作；<br>寄存器数据转换                               | 磁盘控制器、网络适配器                      |
