**Summary：**
> 进程 = 程序 + 数据 + 运行时状态（由内核维护）
> -> 一般称进程映象为：PCB结构与程序和数据的组合
---
### 程序段（代码段）
- 存放可执行指令
- **只读** 
- 多个进程可以共享同一份代码
	-> （比如在linux终端打开多个 `vim`，使用的代码只`/usr/bin/vim`的一个静态文件，程序代码段不会被复制）
---
### 数据段
包括：
- 全局变量
- 静态变量
- 常量区
- ---
### PCB：进程控制模块
#### 主要信息
1. **进程标识符号（PID）**：唯一标识符
2. **进程状态**：就绪态，运行态，阻塞态三选一
3. **处理机状态信息：**
	- 程序计数器（PC）：即将被执行的下一条指令地址
	- 通用寄存器 / PSW / 状态寄存器：上下文数据（暂停时存什么，恢复时就得原封不动还回去的东西）
4. [[调度]]信息：
	- 优先级，
	- 正在等待的事件，
	- 进程[[调度]]所需信息
5. 内存指针：代码和数据的指针，如指向父进程
6. I/O状态信息：I/O请求，设备，文件列表
---
### 内存中的进程
```
| 代码段 (Text) | ← 低地址
| 数据段 (Data) |
| BSS段        |  ← 未初始化全局变量
| 堆 (Heap)    |  ← 向下增长
|              |
|              |
| 栈 (Stack)   |  ← 向上增长
|              |  ← 高地址
```
- 堆栈对冲增长
