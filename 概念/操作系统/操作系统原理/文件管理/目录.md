### 文件共享：软连接与硬链接
#### 硬链接
本质：
- 为同一个 索引节点（inode）**创建多个目录项**
- 即 inode : 文件名 为 1 : n
创建方式：
```bash
ln file1 file2
```
特点：
- 共享同一个 inode 号
- 不会创建数据副本
- 不能用于**目录**
- 仅限于同一个文件系统内部，不能跨越挂载点或磁盘分区
#### 软链接
本质：
- 包含 指向目标文件路径的字符串 的文件
创建方式：
```bash
ln -s file1 file2
```
特点：
- 有独立文件名和 inode
- 删除原文件后，软链接不会失效为独立文件，但会变成 **悬挂链接**，无法再访问目标内容
- 可跨文件系统使用，可使用于目录

### 隐式链接和显式连接
- 隐式：类似链表，链接对用户透明
- 显式：类似集合，认为将存储的磁盘块放进inode中
