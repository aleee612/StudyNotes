### 目录
- 形状：树形
- 本质：文件
- 存储结构：目录项（包含文件名和inode）
#### FCB 文件控制块
| 结构      | 作用                           |
| ------- | ---------------------------- |
| 文件名     | 每一个文件对应一个FCB，文件名用于标识对应的文件控制块 |
| 类型      | 普通文件、目录文件、可执行文件等             |
| 文件权限    | 读、写、执行权限                     |
| 文件大小    | 当前占用字节数                      |
| 文件数据块指针 | 指向文件实际存储的数据块位置，用于找到文件内容      |
| 创建时间    | /                            |
| 修改时间    | 最近一次被修改                      |
| 访问时间    | 最近一次被访问                      |
| 文件拥有者   | 标识文件所属用户                     |
| 链接计数    | 表示有多少目录项指向该文件                |
| 状态标志    | 打开、隐藏、只读                     |
| 索引信息    | 用于快速定位文件磁盘块（如inode中的索引块）     |
- [[文件#文件元信息|inode]] 是FCB的一种具体实现
- 所有FCB的集合构成目录文件
#### 逻辑结构
| 目录结构       | 特点             | 优点                               | 缺点              |
| ---------- | -------------- | -------------------------------- | --------------- |
| 单级目录结构     | 整个系统只有一个目录     | 结构简单                             | 文件容易重名；不便管理     |
| 两级目录结构     | 主文件目录 + 用户文件目录 | 不同用户可重名                          | 用户内部仍可能重名；共享困难  |
| 多级（树形）目录结构 | 目录下可再建子目录      | 层次清晰；便于管理；<br>解决了“文件重名”和“分类管理”问题 | 路径搜索较复杂         |
| 无环图目录结构    | 允许目录/文件共享      | 支持文件**共享**；节省空间                  | 管理复杂；需处理共享和引用计数 |
#### 常见操作
| 操作         | 考点说明                   |
| ---------- | ---------------------- |
| 创建目录       | 分配inode/FCB，并建立目录项     |
| 删除目录       | 通常要求目录为空；涉及目录项和inode回收 |
| 搜索目录       | 根据文件名查找对应目录项           |
| 添加/删除目录项   | 修改“文件名 → inode”映射关系    |
| 路径解析       | 按路径逐级查找目录项             |
| 绝对路径与相对路径  | 路径定位方式的区别              |
| 多级目录结构     | 树形结构；解决文件重名问题          |
| 硬链接        | 多个文件名指向同一inode         |
| 软链接        | 保存目标文件路径的特殊文件          |
| 文件检索过程     | `目录项 → inode → 数据块`    |
| inode与目录关系 | inode不存文件名，目录项才存文件名    |
| 打开文件表      | 管理正在被进程打开的文件信息         |
#### 查找方法
| 查找方式     | 思想            | 特点            |
| -------- | ------------- | ------------- |
| 顺序       | 按目录项依次遍历比较文件名 | 实现简单，但目录大时效率低 |
| Hash（散列） | 对文件名计算哈希定位目录项 | 查找速度快，但实现较复杂  |
#### 物理结构
| 物理实现方式 | 思想               | 优点              | 缺点             |
| ------ | ---------------- | --------------- | -------------- |
| 连续分配   | 文件数据块连续存放在磁盘     | 访问速度快；支持顺序和随机访问 | 易产生外部碎片；文件扩展困难 |
| 链接分配   | 文件数据块离散存放，通过指针连接 | 无外部碎片；扩展方便      | 随机访问效率低；指针占空间  |
| 索引分配   | 建立索引块记录所有数据块地址   | 支持随机访问；扩展方便     | 索引块需要额外空间      |
**隐式链接和显式连接**：
- 隐式：类似链表，链接对用户透明
- 显式：类似集合，认为将存储的磁盘块放进inode中
**单，多，混合索引分配**：
- 单级索引分配：  
    为每个文件建立一个索引块，索引块中直接存放所有数据块地址
    优点：支持随机访问
    缺点：大文件时索引块容量有限
- 多级索引分配：  
    索引块中不直接存数据块地址，而是再指向下一级索引块
    能支持超大文件，但访问时需要多次索引
- 混合索引分配：  
    将直接索引与多级间接索引结合
    小文件优先用直接索引提高效率，大文件再使用一级、二级、三级间接索引扩展容量
