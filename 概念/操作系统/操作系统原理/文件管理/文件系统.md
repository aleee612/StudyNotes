作用：在磁盘之上构建 **逻辑结构**，以 盘块（或簇）为单位进行空间分配和操作
### 文件系统结构
| **组成部分**  | **考点关键词**       | **复习要点**                                    |
| --------- | --------------- | ------------------------------------------- |
| **MBR**   | **主引导、分区表**     | 位于磁盘**0号扇区**<br>负责识别活动分区                    |
| **引导块**   | **分区引导、多系统**    | 每个分区都有，即使没装系统也保留<br>负责启动该分区内的OS             |
| **超级块**   | **全局配置、Meta信息** | 包含**块大小、空闲块/i节点数量**<br>挂载时必须最先读入内存          |
| **空闲块管理** | **分配策略**        | 常见实现：**位示图**（考计算）、**空闲链表**、成组链接法            |
| **i 节点区** | **文件索引、属性**     | 包含文件属性（大小、权限）和磁盘块指针<br>**不含文件名**            |
| **根目录**   | **检索起点**        | 查找文件的入口<br>在 UNIX 中，根目录通常固定在特定的 i 节点号（如 2号） |
| **数据区**   | **实际存储**        | 占据磁盘绝大部分空间<br>存放目录项数据和文件内容                  |

### 外存空闲管理
| 方法    | 核心思想           | 数据结构         | 分配方式       | 回收方式    | 优点                   | 缺点                  | 常考点               |
| ----- | -------------- | ------------ | ---------- | ------- | -------------------- | ------------------- | ----------------- |
| 空闲表法  | 记录连续空闲区间       | 空闲表（起始块号+长度） | 首次/最佳/最坏适应 | 合并相邻空闲区 | 适合连续文件分配；连续空间查找快     | 外碎片问题明显             | 首次适应、最佳适应等算法      |
| 空闲链表法 | 将空闲块链接成链       | 链表           | 从链头取空闲块    | 插回链表    | 实现简单                 | 查找连续空间困难；效率一般       | 分为“空闲盘块链”和“空闲盘区链” |
| 位示图法  | 用二进制位表示磁盘块是否空闲 | 位示图（0/1数组）   | 扫描位图找空闲块   | 修改对应位   | 管理直观；适合随机访问；容易统计空闲空间 | 大磁盘时位图较大；查找连续空闲块可能慢 | 位号与盘块号转换；空间计算     |
| 成组链接法 | 把若干空闲块号分组存储    | 栈 + 分组链      | 从栈顶取块      | 块满时成组链接 | 兼顾效率与空间；UNIX常用       | 实现较复杂               | 栈空/栈满时如何切换组       |
#### 空闲表法
- 数据结构：链表
	1. 链表的头节点保存第一个空闲块的位置
	2. 每个节点的 **next** 指针指向 **下一个空闲块**，形成一条从头到尾依次链接的空闲块序列
#### 位图法
- 数据结构：二进制数组
	1. 位图中的每一位对应磁盘上一个固定大小的数据块，通过该位的取值即可快速判断该块的使用状态
	2. 使用 0, 1 判断是否被使用
计算公式：
- 对于`map[i,j]` 
- 第`i`行、 第`j`列（行、列编号从1开始）
- 每行包含`n`位
- 盘块号`b` 
$$b=n(i-1)+j$$
$$i=(b-1)/n+1$$
$$j=(b-1)\%n+1$$
#### 成组链接法
- 数据结构：栈+链表
	1. 使用一个磁盘块来保存若干空闲盘块号
	2. 通过链表把这些块组织起来
### 虚拟文件系统 VFS
- 屏蔽不同存储介质（本地磁盘、网络共享、光盘等）和各类文件系统的实现细节
- 相当于硬件层面的封装

| **对象名称**        | **核心职能**          | **关键考点 (必背)**                       |
| --------------- | ----------------- | ----------------------------------- |
| **超级块对象**       | 描述**整个文件系统**的信息   | 对应磁盘上的“超级块”；存储块大小、状态、挂载信息           |
| **索引节点对象inode** | 描述**单个文件**的物理属性   | **唯一性**：每个文件有且仅有一个 inode 号；不含文件名    |
| **目录项对象**       | 描述**文件的逻辑结构**（路径） | 维护文件名与 inode 的映射关系；**只存在于内存**中，不存磁盘 |
| **文件对象**        | 描述**进程与已打开文件**的交互 | 存储文件指针（offset）、打开标志；每个进程打开一次就会创建一个  |
### 文件系统挂载
- 把一个独立的文件系统与文件树关联起来，使得该文件系统中的文件和目录能够被正常访问和管理
- 步骤：
	1. 识别存储设备  
	2. 选择挂载点 ：挂载点本身必须是一个已存在且为空的目录
	3. 执行挂载操作：OS调用相应的挂载函数，将外部文件系统的根目录映射到挂载点
	4. 管理与维护
