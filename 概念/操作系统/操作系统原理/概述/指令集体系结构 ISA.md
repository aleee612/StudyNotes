> **Summary：**
> **ISA 是 CPU 对程序员暴露的“接口”**  
> 就像 API，但这是 **硬件给软件的 API**。

- 编译器、操作系统、汇编代码 **只关心 ISA**，
- **不关心 CPU 内部到底是流水线还是乱序执行**。
---
### 和软件的关系
- **汇编语言**：是 ISA 的“文本表示”
- **编译器**：把高级语言翻译成某 ISA 的指令
- **操作系统**：运行在 ISA 之上
- **跨平台难**：因为 ISA 不一样（x86 ≠ ARM）
- ---
### 具体规定什么？
- **指令集**：  
    `add`、`mov`、`load`、`jump` 能不能用、怎么写
- **寄存器**：  
    有哪些寄存器（如 x86 的 RAX、ARM 的 X0）
- **数据类型**：  
    int、float、位宽是多少
- **寻址方式**：  
    立即数、寄存器寻址、内存寻址
- **异常 / 中断模型**
- **内存模型（可见行为）**

> **不包括** CPU 内部怎么实现（那是“微体系结构”的事）
---
### ISA vs 微体系结构（Microarchitecture）

| 项目        | ISA         | 微体系结构      |
| --------- | ----------- | ---------- |
| 面向谁       | 软件 / 编译器    | 硬件工程师      |
| 关注点       | 指令“是什么、怎么用” | 指令“怎么跑得更快” |
| 是否可见      | ✅ 可见        | ❌ 不可见      |
| 会不会影响程序兼容 | ✅ 会         | ❌ 不会       |

- 同一 ISA，可以有无数种 CPU 实现
