### IEEE 浮点数标准
**单精度**：
![[Pasted image 20260118164751.png|558]]
- 浮点数表示为：$(−1)^s×1.frac×2^{exp−127}$ 
**双精度**：
![[Pasted image 20260118164832.png|565]]
- 浮点数表示为：$(−1)^s×1.frac×2^{exp−1023}$ 
> s 为 **符号位**
> 翻译：**正负号 × 一个 1 点几的数 × 2 的多少次方**
##### 例子：单精度算 5.75
第一步：转成二进制
`5.75 = 101.11（二进制）`
第二步：科学计数法（只留一个 1 在前面）
`101.11 = 1.0111 × 2²`
第三步：三部分填进去
- **符号位**：  正数 → `0`
- **指数位**：  2 + 127 = 129  129 = `10000001`
- **尾数**：  小数部分 `.0111`  补到 23 位：`01110000000000000000000`
最终长这样：
`0 | 10000001 | 01110000000000000000000`

### 异常值
| 类型   | 阶码（exp）   | 尾数（fraction）     | 有效值公式                  |
| ---- | --------- | ---------------- | ---------------------- |
| 正常值  | 非全 0、非全 1 | f                | $±1.f×2^{e−偏移量}$       |
| 非正常值 | 全 0       | f （必须非 0，否则是 0 ） | $±0.f×2^{1-偏移量}$       |
| ±0   | 全 0       | 全 0              | ±0                     |
| ±∞   | 全 1       | 全 0              | ±∞                     |
| NaN  | 全 1       | 非 0              | Not a Number（结果非法或未定义） |

### 规格化
- 根据尾数表示形式，确定补码规格化方式
**补码尾数模型**（符号位存储在尾数中）：
	1. 尾数补码：`1.0xxx` 为规格化负数，`0.1xxx` 为规格化正数
	2. 尾数原码：`1.1xxx` 为规格化负数，`0.1xxx` 为规格化正数
	3. 默认不省略最高位`1`，存储尾数为`01xxx`，`11xxx`，`10xxx`
**IEEE754**（符号位在整个浮点数首位）：
	4. `1.xxx` 为规格化数
	5. 尾数存储时默认省略最高位`1`，存储的仅为`xxx`

### 对阶
> 小对大，尾右移
- 需注意的是，由于在对阶中永远是小阶码对大阶码，因此不可能出现阶码减小/尾数左移的情况

---
### 舍入
**对阶舍入：**
- 右移时，低位的部分会被移出尾数存储范围
- 这部分就是 **被丢掉的位** → 需要**舍入**来尽量减少误差
