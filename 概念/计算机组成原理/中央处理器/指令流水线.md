### 五段流水结构
| 流水段 | 英文  | 功能          |
| --- | --- | ----------- |
| 取指  | IF  | 取指令、修改 PC   |
| 译码  | ID  | 分析指令、取寄存器   |
| 执行  | EX  | 算术逻辑运算、形成地址 |
| 访存  | MEM | 访问数据存储器     |
| 写回  | WB  | 结果写入寄存器     |
口诀：
```text
取 → 译 → 执 → 访 → 写
```

#### 顺序执行与流水执行比较
| 项目         | 顺序执行     | 流水执行       |
| ---------- | -------- | ---------- |
| 并行性        | 无        | 有          |
| 单条指令时间     | $k·t$    | $k·t$      |
| 吞吐率        | 低        | 高          |
| 完成 n 条指令时间 | $nkt$    | $(k+n−1)t$ |
| 极限吞吐率      | $1/(kt)$ | $1/t$      |
| 极限加速比      | $1$      | $k$        |
> 流水线提高吞吐率，不缩短单条指令执行时间
#### 使用流水线的条件
##### 基本条件
| 条件          | 含义          |
| ----------- | ----------- |
| 指令能够分解成若干阶段 | 每个阶段完成不同功能  |
| 各阶段功能相对独立   | 前后阶段可并行工作   |
| 各段所需时间尽可能接近 | 避免某一段成为瓶颈   |
| 各阶段连续提供数据   | 保证流水线不断流    |
| 阶段之间有缓冲寄存器  | 保存中间结果，实现并行 |
##### 各流水段时间要求
| 情况      | 结果      |
| ------- | ------- |
| 各段时间相等  | 效率最高    |
| 各段时间接近  | 效率较高    |
| 某一段明显过长 | 吞吐率受限   |
| 某一段明显过短 | 硬件利用率降低 |
#### 目的
| 项目   | 内容           |
| ---- | ------------ |
| 根本目的 | 提高系统吞吐率      |
| 本质   | 时间并行技术       |
| 提高对象 | CPU利用率、硬件利用率 |
| 不提高  | 单条指令执行时间     |
| 不改变  | 单条指令的执行过程    |
#### 流水线重要公式
| 名称     | 公式                       | 极限情况        |
| ------ | ------------------------ | ----------- |
| 流水周期   | $t=max(各段时间)$            | ——          |
| 顺序执行时间 | $T_顺=n·k·t$              | ——          |
| 流水执行时间 | $T_流=(k+n−1)t$           | $n≫k_时≈nt$  |
| 实际吞吐率  | $T_P=n/T_流$              | ——          |
| 最大吞吐率  | $T_{Pmax}=1/t$           | ——          |
| 加速比    | $S=T_顺/T_流= n·k/(k+n−1)$ | $S_{max}=k$ |
| 效率     | $η=S/k=n/(k+n−1)$        | $η_{max}=1$ |
#### 流水线周期确定
| 各段时间   | IF  | ID  | EX  | MEM | WB  |
| ------ | --- | --- | --- | --- | --- |
| 时间(ns) | 100 | 80  | 60  | 120 | 90  |
则：

| 项目   | 结果     |
| ---- | ------ |
| 流水周期 | 120 ns |
| 原因   | 取最大值   |

### 流水线冒险
| 类型   | 原因     | 例子           | 解决方法         |
| ---- | ------ | ------------ | ------------ |
| 结构冒险 | 争夺硬件资源 | IF、MEM同时访问主存 | 增加资源、暂停      |
| 数据冒险 | 数据尚未产生 | 后一条指令使用前一条结果 | 旁路、暂停、编译优化   |
| 控制冒险 | 转移地址未知 | if、jmp、beq   | 分支预测、暂停、延迟分支 |
口诀：
```text
结构抢资源
数据等结果
控制等方向
```

| 类型      | 英文      | 顺序关系           | 是否真相关   |
| ------- | ------- | -------------- | ------- |
| **读后写** | **RAW** | **Write→Read** | **真相关** |
| 写后读     | WAR     | Read→Write     | 假相关     |
| 写后写     | WAW     | Write→Write    | 输出相关    |
#### 数据冒险解决方法
| 方法       | 原理              | 是否停顿 |
| -------- | --------------- | ---- |
| 插入气泡     | 暂停流水线           | 是    |
| 数据旁路（转发） | EX/MEM结果直接送后续阶段 | 否    |
| 编译优化     | 调整指令顺序          | 一般否  |
#### 控制冒险解决方法
| 方法    | 原理         |
| ----- | ---------- |
| 暂停流水线 | 等待分支结果     |
| 分支预测  | 预测跳转方向     |
| 延迟分支  | 将无关指令放入延迟槽 |
> 若预测失败则 **Flush**（冲刷流水线）清除错误指令

### 超流水线与超标量 
| 类型    | 本质         | 提高方式   |
| ----- | ---------- | ------ |
| 超流水线  | 增加流水段数     | 提高时钟频率 |
| 超标量   | 一个周期发射多条指令 | 提高并行度  |
| 普通流水线 | 一个周期发射一条指令 | ——     |
例如：

| 类型     | 发射宽度 |
| ------ | ---- |
| 普通流水线  | 1    |
| 双发射超标量 | 2    |
| 四发射超标量 | 4    |

### 时空图规律
- k 段流水
- n 条指令
则：

| 项目          | 公式         |
| ----------- | ---------- |
| 总周期数        | $k+n−1$    |
| 第一条指令完成时间   | $kt$       |
| 第 n 条指令完成时间 | $(k+n−1)t$ |
| 稳定状态        | 每周期完成一条指令  |

### 几个容易混淆的概念
| 名词             | 含义            |
| -------------- | ------------- |
| Bubble（气泡）     | 插入空周期         |
| Stall（停顿）      | 暂停流水线         |
| Forwarding（旁路） | 数据不经过 WB，直接转发 |
| Flush（冲刷）      | 清除错误进入流水线的指令  |
| Hazard（冒险）     | 流水线冲突         |
