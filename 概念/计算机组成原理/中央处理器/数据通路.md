### 组成
| **维度 / 考点** | **组合逻辑元件** （操作元件）                               | **时序逻辑元件** （状态元件）                 |
| ----------- | ----------------------------------------------- | --------------------------------- |
| **根本定义**    | 输出**仅**取决于当前输入的逻辑组合                             | 输出不仅取决于当前输入，还取决于**过去的状态**（具有记忆功能） |
| **核心特征**    | **无状态、无记忆**、无反馈回路                               | **有状态、有记忆**、包含反馈回路、依赖时钟信号         |
| **典型代表**    | 门电路（与/或/非）<br>加法器（ALU）<br>译码器、编码器<br>多路选择器（MUX） | 触发器（如D触发器）<br>寄存器<br>计数器、状态机      |
| **时间特性**    | **延迟**：信号通过门电路需要时间                              | **建立时间/ 保持时间**：时钟边沿前后数据必须稳定的时间    |
### 分类
#### 部件连接方式划分
| **维度 / 考点** | **总线式结构**                                                     | **分散式结构 / 专用线路连接 **                                |
| ----------- | ------------------------------------------------------------- | -------------------------------------------------- |
| **核心定义**    | 所有或多个部件都连接到**一组公共的传输线（总线）**上，部件间通过总线交换数据                      | 部件之间**没有公共总线**，而是根据数据流向，在需要通信的部件之间**直接连线**         |
| **细分类别**    | **内部单总线**：所有部件接在一条总线上<br>**多总线（双/三总线）**：按功能或速度分组（如ALU输入/输出分开） | **点对点专用连接**：完全定制<br>**基于MUX（多路选择器）连接**：利用MUX来筛选输入源 |
| **硬件复杂度**   | **低，结构简单**<br>连线少，规整，容易扩展新部件                                  | **高，连线繁杂**<br>随着部件增加，连线数量和MUX的数量会呈指数级增长            |
| **数据并行度**   | **低**<br>在单总线上，**同一时刻只能有一个部件发送数据**，否则会发生冲突（总线争用）              | **高**<br>不同的部件对之间可以**同时进行**数据传输，互不干扰               |
| **性能/速度**   | **较慢**<br>因为数据需要分时分步通过总线，容易成为性能瓶颈                             | **快**<br>专用线路延迟小，且支持高度并行，是高性能CPU（如流水线CPU）的首选       |
#### 时序组织方式划分
| **维度 / 考点** | **单周期数据通路**                                          | **多周期数据通路 **                                       |
| ----------- | ---------------------------------------------------- | -------------------------------------------------- |
| **核心定义**    | 每一条指令的执行**都在一个时钟周期内**完成                              | 将一条指令的执行过程**拆分为多个较小的步骤**，每个步骤占用一个时钟周期              |
| **时钟周期定义**  | 取决于**执行最慢的那条指令**（通常是访存指令 `Load`）所需要的时间               | 取决于**耗时最长的那一个子步骤**（如ALU计算或访存）                      |
| **硬件复用性**   | **不复用**<br>同一个部件在一个周期内只能用一次，因此需要独立的指令存储器和数据存储器，多个加法器 | **高度复用**<br>同一个部件在不同周期可以重复使用                       |
| **内部寄存器**   | 只需要程序计数器（PC）和通用寄存器组，不需要指令间的暂存寄存器                     | 需要大量的**暂存寄存器**（如IR、MDR、A、B、ALUOut），用于在步骤之间保存中间结果   |
| **性能/效率**   | **低**<br>对于执行快的指令（如 `Add`），也必须等待和最慢指令一样长的周期，造成时间浪费   | **较高**<br>指令按需占用周期（简单的指令2-3个周期，复杂的4-5个周期），平均执行时间更短 |
