> 操作系统中的中断：[[中断]] 
### 异常
| 类型        | 含义                 | 是否由当前指令引起 | 返回位置           |
| --------- | ------------------ | --------- | -------------- |
| 故障（Fault） | 指令执行过程中检测到异常，可恢复   | 是         | 返回**当前**指令重新执行 |
| 自陷（Trap）  | 人为有意设置的异常，如系统调用、断点 | 是         | 返回**下一条**指令    |
| 终止（Abort） | 严重错误，如硬件故障、校验错     | 否         | 一般不能恢复，**不返回** |
#### 响应过程
1. 关中断：[[IF中断允许位|IF]] 状态调整
2. 保存断点和程序状态字（PSW）
3. 识别事件类型
4. 查中断向量表，获得服务程序入口地址
5. 转入中断/异常处理程序
6. 执行处理程序
7. 恢复现场
8. 开中断
9. 返回被中断程序

---
### 中断
#### 分类
| 分类依据       | 类型          | 特点                      |
| ---------- | ----------- | ----------------------- |
| **能否屏蔽**   | 可屏蔽中断（INTR） | 可由中断允许触发器控制             |
|            | 不可屏蔽中断（NMI） | 不能被屏蔽，优先级最高             |
| **来源**     | 外中断         | 来自 CPU 外部，如 I/O 完成、时钟中断 |
|            | 内中断（异常）     | 来自 CPU 内部，如除零、缺页、系统调用   |
| **是否主动产生** | 硬件中断        | 由硬件设备发出                 |
|            | 软件中断        | 由指令产生，如系统调用             |
#### 中断响应过程
1. 中断请求
2. 中断判优
3. 中断响应（关中断）
4. 保存断点和现场
5. 查中断向量表
6. 执行中断服务程序（ISR）
7. 恢复现场
8. 开中断
9. 中断返回

---
### 异常与中断区别
| 项目        | 异常         | 中断            |
| --------- | ---------- | ------------- |
| 来源        | CPU 内部     | CPU 外部        |
| 是否与当前指令有关 | 有关         | 无关            |
| 是否同步      | 同步事件       | 异步事件          |
| 是否可预测     | 一般可预测      | 不可预测          |
| 典型例子      | 除零、缺页、系统调用 | 时钟中断、I/O 完成中断 |
#### 常见例子
- **故障（Fault）**：缺页、除法溢出
- **自陷（Trap）**：系统调用、调试断点
- **终止（Abort）**：硬件故障、奇偶校验错
- **可屏蔽中断**：键盘输入、磁盘完成
- **不可屏蔽中断**：电源故障、存储器校验错
