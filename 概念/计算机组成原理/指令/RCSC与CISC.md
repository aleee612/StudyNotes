
| 对比项         | RISC（精简指令集）                 | CISC（复杂指令集）         |
| ----------- | --------------------------- | ------------------- |
| 核心思想        | 简化指令，提高执行效率                 | 用复杂指令减少程序条数         |
| 指令数量        | 少                           | 多                   |
| 指令长度        | 固定长度居多                      | 可变长度居多              |
| 指令格式        | 规整、统一                       | 多样、复杂               |
| 寻址方式        | 少                           | 多                   |
| 指令执行周期      | 大多单周期                       | 多周期较常见              |
| 控制方式        | 硬布线控制居多                     | 微程序控制居多             |
| CPI（平均指令周期） | 较低                          | 较高                  |
| 时钟频率        | 容易提高                        | 较难提高                |
| 流水线实现       | 容易（必须实现）                    | 较困难（但可以通过一定方式实现）    |
| 编译器依赖       | 强                           | 相对弱                 |
| 指令功能        | 简单，一条只做少量工作                 | 一条可完成复杂操作           |
| 内存访问        | Load/Store结构，<br>仅访存指令能访问内存 | 大部分指令可直接访存          |
| **通用寄存器数量** | **多**                       | **较少**              |
| 硬件复杂度       | 较低                          | 较高                  |
| 软件复杂度       | 较高（依赖编译器优化）                 | 较低                  |
| **代码长度**    | **较长**                      | **较短**              |
| **执行效率**    | **高频+流水线下高**                | **单条指令功能强**         |
| 功耗          | 通常较低                        | 通常较高                |
| 面积          | 较小                          | 较大                  |
| 设计目标        | 提高执行速度                      | 提高编程便利性             |
| 典型代表        | ARM、MIPS、RISC-V、SPARC       | x86、VAX             |
| 现代趋势        | 大量应用于移动端、嵌入式                | PC服务器仍大量使用          |
| 现代融合趋势      | 实际上两者界限模糊                   | x86内部也会转微操作（类似RISC） |
