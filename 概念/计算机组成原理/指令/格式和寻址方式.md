#### 定长和变长指令
**定长指令集**：所有指令的长度完全相同  
    → 典型架构：ARM
    → 优点：解码简单、高效  
    → 缺点：指令中可能出现浪费空间的无效字段
- 对于地址字段短于定长的指令，则使用无效字段填充
- 若长于，则使用 多字指令 或 简介寻址（使用内存字段替代真正操作数地址）
**变长指令集**：不同指令具有不同长度
    → 典型架构：x86
    → 优点：编码更紧凑，能支持更复杂操作  
    → 缺点：解码过程复杂，需要准确识别指令边界
- 识别指令边界：使用指令前缀
##### 操作码扩展编码
在定长指令中，地址越长，则操作码越短
一般认为：16位指令字中，
	三地址指令 4 位操作码  
	二地址指令 8 位操作码  
	一地址指令 12 位操作码
##### 计算公式：
假设地址码长度为 $n$ 位：
$$\text{下一级指令最大条数} = (\text{本级状态总数} - \text{本级已用条数}) \times 2^n$$

| **指令级别** | **操作码总空间 (A)**   | **已用条数 (B)** | **剩余空间 (A-B)** |
| -------- | ---------------- | ------------ | -------------- |
| **三地址**  | $2^{(L - 3n)}$   | 题目给出         | $R_3$          |
| **二地址**  | $R_3 \times 2^n$ | 题目给出         | $R_2$          |
| **一地址**  | $R_2 \times 2^n$ | 题目给出         | $R_1$          |
| **零地址**  | $R_1 \times 2^n$ | 最大可用数        | -              |

### 寻址方式
#### 指令寻址 
| 寻址方式 | 含义                                                | 优点                          | 缺点/适用场景                                                  | 计算有效地址 (EA) 示例           |
| ---- | ------------------------------------------------- | --------------------------- | -------------------------------------------------------- | ------------------------ |
| 顺序寻址 | 通过程序计数器（PC）自动加 n（n 为当前指令占用的存储单元数），自动读取下一条紧邻的指令。   | 结构最简单，程序按代码编写顺序默认执行，效率高。    | 大部分普通指令（如算术运算、数据传送等非跳转指令）。                               | 执行完当前指令后：PC←PC+n         |
| 跳跃寻址 | 通过转移类指令（如跳转、条件分支、子程序调用）直接修改 PC 的值，使得下一条指令不再是顺序读取。 | 能够实现复杂的程序逻辑控制（如循环、判断、函数调用）。 | 分支语句（if-else）、<br>循环语句（for/while）、<br>函数调用（call/return）。 | 执行跳转指令：JMP 2000此时PC←2000 |
#### 数据寻址
| 寻址方式        | 含义                         | 操作数位置 | 优点                   | 缺点/适用场景        | 计算有效地址 (EA) 示例                                   |
| ----------- | -------------------------- | ----- | -------------------- | -------------- | ------------------------------------------------ |
| **立即寻址**    | 指令中直接给出操作数                 | 指令本身  | 取数快，简单               | 不能修改操作数，只能用于常数 | `EA` 不需要，操作数 = 指令中值                              |
| **直接寻址**    | 地址字段给出操作数的内存地址             | 内存    | 简单，直观                | 地址字段长，需要更多位    | `EA = 地址字段`                                      |
| **间接寻址**    | 地址字段指向一个存储真正地址的内存单元        | 内存    | 节省地址位，可访问大地址空间       | 访问慢，多次内存访问     | `EA = M[地址字段]`                                   |
| **寄存器寻址**   | 操作数在寄存器中                   | 寄存器   | 访问快                  | 寄存器数量有限        | `EA = Rn`（寄存器编号）                                 |
| **寄存器间接寻址** | 地址在寄存器里，寄存器存放有效地址          | 寄存器   | 快，适合遍历数组             | 寄存器有限          | `EA = [Rn]`（寄存器内容作为地址）                           |
| **相对寻址**    | 地址 = PC + 指令中偏移            | 内存    | 支持分支，程序可移动           | 偏移范围受限         | `EA = PC + offset`                               |
| **基址寻址**    | 地址 = 基址寄存器 + 偏移            | 内存    | 支持大内存区，灵活            | 需占用寄存器         | `EA = BR + offset`                               |
| **变址寻址**    | 地址 = 变址寄存器 + 偏移（常用于**数组**） | 内存    | 支持数组访问               | 寄存器有限          | `EA = XR + offset`<br>`操作数 = M[EA]`*变址寻址只访问一次内存* |
| **基址+变址寻址** | 地址 = 基址寄存器 + 变址寄存器 + 偏移    | 内存    | 支持复杂数据结构             | 复杂，寄存器占用多      | `EA = BR + XR + offset`                          |
| **堆栈寻址**    | 地址 = 栈顶指针 SP 所指地址          | 栈（内存） | 适合函数调用、中断保护、参数传递；指令短 | 只能访问栈顶附近数据，不灵活 | `EA = (SP)`、`PUSH AX`、`POP AX`                   |
- 变址寄存器：专门计算具体偏移用（算上元素大小）
- 基址寄存器：存放数据结构起始位置

##### 访问内存次数
| 寻址方式     | 内存访问次数  |
| -------- | ------- |
| 直接寻址     | 1 次     |
| 变址寻址     | 1 次     |
| 寄存器间接    | 1 次     |
| **间接寻址** | **2 次** |
