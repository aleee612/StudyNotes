### BST 二叉排序树
#### 特性
1. 左子树非空，则左子树上所有结点的关键字均小于根结点的关键字
2. 若右子树非空，则右子树上所有结点的关键字均大于根结点的关键字
3. 左右子树也分别是一棵二叉排序树
#### 查找
> 从根结点开始，逐层向下比较并选择左/右子树
```cpp title=查找
BSTNode *BST_Search(BiTree T,ElemType key){ 
	while(T!=NULL&&key!=T->data){ //若树空或等于根结点值，则结束循环 
		if(key<T->data)	T=T->1child; //小于，则在左子树上查找 
		else T=T->rchild; //大于，则在右子树上查找 
	}
	return T;
}
```
#### 插入
1. 若原二叉排序树为空，则将新结点作为根结点直接插入
2. 否则，从根结点开始比较
	1. 若关键字 k 小于当前结点的关键字，则在左子树中继续查找并插入
	2. 若 k 大于当前结点的关键字，则在右子树中继续查找并插入
```cpp title=插入
int BST_Insert(BiTree &T,KeyType k){
	if(T==NULL){ //原树为空，新插入的记录为根结点 
		T=(BiTree)malloc(sizeof(BSTNode)); 
		T->data=k; 
		T->1child=T->rchild=NULL; 
		return 1; //返回1,插入成功 
	} 
	else if(k==T->data) //树中存在相同关键字的结点，插入失败 
		return 0; 
	else if(k<T->data) //插入T的左子树 
		return BST_Insert(T->1child,k); 
	else //插入T的右子树 
		return BST_Insert(T->rchild,k);
}
```

### AVL 平衡二叉树
#### 定义
作用：避免树的高度增长过快而导致二叉排序树性能下降
操作：在插入和删除结点时，保证任意结点的**左右子树高度差的绝对值不超过1** 
1. 每当插入一个结点后，沿查找路径向上检查各祖先结点是否因该操作而失去平衡
2. 若存在**失衡结点**，则定位插入路径上离新结点**最近**、且平衡因子绝对值大于 1 的结点A，以A为根的子树即为最小不平衡子树
3. 在保持二叉排序树特性的前提下，通过**旋转调整**该子树的结构，使其重新达到平衡
#### 旋转
> 哪个地方太重，就把它拉回来，AVL 就是在让树重新平衡

> [!note]- 旋转总览
> - 插入（或删除）节点
> - 从该节点向上找，找到**第一个失衡节点 A**
> - 找到 **A → B → C** 三个关键节点
> - 判断是否拐弯（LL、RR、LR、RL）
> - **只调整 A 为根的那棵子树**
> - 把调整后的子树接回原树即可

1. 找失衡点（平衡因子大于1，平衡因子 = 左子树高度 − 右子树高度）
2. 判断拐弯没有
	认为形如：
	```
	A
	 \
	  B
	   \
	    C
	```
	的树为未拐弯，
	形如：
	```
   A
  /
B
  \
   C
	```
	的树为拐弯
3. 未拐弯：一次旋转即可
	如左左，
	```
	    A
   /
  B
 /
C
	```
	右旋变成：
	```
	   B
   / \
  C   A
	```
4. 拐弯：需要两次旋转
	```
	   A
   /
  B
   \
    C
	```
	1. 先左旋 B
	```
     A
    /
   C
  /
B
	```
	2. 再右旋 A
	```
	   C
   / \
  B   A
	```
> **旋转不会破坏二叉搜索树的中序序列**
- 实际上考试时真正参与判断旋转类型的，通常只有三个关键节点：
	- **A**：第一个失衡节点
	- **B**：A 的较高子树根
	- **C**：B 的较高子树根
- 其余节点只是作为整体子树一起移动，不需要逐个分析
