### 最小生成树
> 一个连通图的生成树包含了图的所有顶点，并且仅包含尽可能少的边
> 具有**最小总权重**的生成树称为图G的最小生成树
#### Prim
![[Pasted image 20260713144942.png]]
1. 从图中任取一个顶点加入树 T ，此时 T 仅包含该顶点
2. 选择一个与当前 T 中顶点**集合**距离**最近**的顶点，并将该顶点及其相连的最小权值边加入T
3. 每执行一次此操作，T 中的顶点数和边数各增加1
4. 重复这一过程，直到图中所有顶点都被并入T为止
5. 最终得到的T即 为最小生成树，且其中必然含有n-1条边
```cpp title=Prim
void Prim(G,T){ 
	T=0; //初始化空树 
	U={W}; //添加任意一个顶点w 
	while((V-0)!=Q){ //若树中不含全部顶点 
	//设(u,v)是使ueU与v∈(V-U),且权值最小的边； 
	T=TU{(u,v)}; //边归入树 
	U=0U{v}; //顶点归入树 
	}
}
```
- 时间复杂度：$O(V^2)$ ，与边数无关
- 适用于稠密图
#### Kruskal
> 按边的权值递增次序选择合适的边来构造
![[Pasted image 20260713145621.png]]
1. 图 $T=\{V,0\}$ 包含全部 `n` 个顶点，但不含任何边，每个顶点**自成一个连通分量**
2. 按照边的权值**从小到大**的顺序，依次考察各条边：
	1. 如果当前边连接的两个顶点属于 T 中**不同**的连通分量(可通过[[并查集→最小生成树|并查集]]判断)，则将该边**加入 T** 
	2. 否则，舍弃此边，继续考察下一条权 值最小的边
3. 重复这一过程，直到所有顶点都属于同一个连通分量，此时得到的 T 即为最小生成树
```cpp title=kruskal
void Kruskal(V,T){ 
	T=V; //初始化树T,仅含顶点 
	numS=n; //连通分量数 
	while(nums>1){ //若连通分量数大于1 从E中取出权值最小的边(v,u); 
		if(v和u属于T中不同的连通分量){ 
			T=T∪{(v,u)}; //将此边加入生成树中 
			numS--; //连通分量数减1 
		}
	} 
}
```
- 时间复杂度：$O(E·log_2E)$ ，与点数无关
- 适用于稀疏图（顶点多边少）

### 最短路径
> 带权路径**长度最小**的路径(可能存在多条)称为最短路径
- 求 某个顶点 到 其余各顶点 **1 : n**
	=>Dijkstra
- 所有 顶点对 之间 **1 : 1**
	=>Floyd
#### Dijkstra
> 使用一个集合 `S` 记录已确定最短路径的顶点
1. 将源点（顶点0）放入 `S`
2. 每当将一个新顶点 `i` 加入 `S` 后，需更新源点到所有尚未确定最短路径的顶点（集合V-S中的 顶点）的当前最短路径长度
###### 数据结构：
1. `final[]`：标记各顶点是否已找到最短路径（是否属于集合S)
2. `dist[]`：记录从**源点**到各顶点的当前最短路径**长度**
	1. 初始化时，若存在从源点到顶点`i`的 直接边，则`dist[]`为该边的权值
	2. 否则置为 $∞$ 
3. `path[]`: `path[i]`存储从源点到顶点 `i` 的最短路径，算法结束后可通过`path[]`数组回溯，重构出完整的最短路径
###### 步骤
![[Pasted image 20260713154015.png]]
> $arcs[i][j]$ 为有向边 $<i,j>$ 的权值
1. 集合 `S={0}`，`dist[]` 的初始值为 $dist[i]=arcs[0][i],i=1,2,⋯,n-1$
2. **选择最短路径顶点：** 从不在 S 中的顶点集合 $(V-S)$ 中选出顶点 `j`，使得 `dist[]`**最小**，此时顶点 `j` 即为当前从源点出发的最短路径的终点，将其加入集合 `S`
3. **松弛操作：** 
	对每个从顶点 `j` 出发的邻接顶点$k(arcs[j][k]≠∞)$，
	若 $dist[j]+ arcs[j][k]< dist[k]$，
	则更新 $dist[k]=dist[j]+arcs[j][k]$ 
4. 重复步骤 2 - 3共 `n-1` 次，直至所有顶点都包含在集合 S 中
###### 特点
- 时间复杂度：$O(V^2)$ ，与边数无关
- 不适用于存在负权边的图
#### Floyd
1. 迭代生成一系列 n 阶方阵$A^{(-1)},A^{(0)},⋯,A^{(k)},⋯,A^{(n-1)}$
	其中$A^{(k)}[i][j]$表示从顶点 $v_i$ 到 $v_j$  、仅允许使用编号不超过 `k` 的顶点作为中间顶点时的最短路径长度
2. 初始时 $(k=-1)$，若存在从 $v_i$, 到 $v_j$ 的直接边，则$A^{(-1)}[i][j]=0$ 为该边的权值；否则设为∞
3. 特别地，对所有 `i` ,令$A^{(-1)}[i][j]=0$
4. 随后，依次考虑顶点 $k(k=0,1,⋯,n-1)$ 作为中间顶点
5. 若经由v的路径比当前记录的路径更短，则更新对应的距离
![[Pasted image 20260713154752.png]]
###### 特点
- 时间复杂度：$O(V^3)$ ，与边数无关
- 在中等规模图上实际运行效率较高
- 允许图中存在负权边，但不允许存在负权环

### 有向无环图描述表达式
> 表示含有公共子表达式的代数表达式
![[Pasted image 20260713155345.png]]
- 如$((a+b)*(b*(c+d))+(c+d)*e)*((c+d)*e)$
- 子表达式在多个位置被复用（如$(c+d)*e$）
- 若采用有向无环图，公共子表达式只需存储一次，从而显著节省存储空间
### 拓扑排序
> [[拓扑排序]] 
#### 前提
1. 每个顶点恰好出现一次
2. 若存在从顶点 A 到 B 的路径，则在排序后的序列中， B 必须排在 A 之后
#### 步骤
1. 从AOV 网中选择一个没有前驱（入度为0）的顶点并输出之
2. 从网中删除该顶点和所有以它为起点的有向边
3. 重复 1 - 2，直到当前 AOV 网为**空**或无法找到**新的入度为0**的顶点为止，后一种情况表明有向图中必然存在环
#### 特点
1. 图中可能存在多个入度为 0 的顶点，随机选择一个输出，因此拓扑序列可能**不唯一**
2. 根据步骤 3 可知，拓扑排序可判断图中有环
3. 时间复杂度：
	1. 邻接表：$O(V+E)$
	2. 邻接矩阵：$O(V^2)$ 
### 关键路径AOE网
> 条件：带权有向图
> 表示：
> 	顶点 => 事件
> 	有向边 => 活动
> 	边上的权值 => 完成该活动所需的开销(如时间)
> 则：称这种图为用边表示活动的网络，简称 AOE网
#### AOE网和AOV网：
- 同：是有向无环图
- 区别：
	- AOE 网的边带有权值，代表活动及其持续时 间，而顶点表示事件
	- AOV网的边无权值，仅用于表示顶点(活动)之间的前后关系
#### 特点
1. 只有在某顶点所代表的事件**发生后**，从该顶点**出发**的各有向边所代表的活动才能开始 
2. 只有当**进入**某顶点的各有向边所代表的活动都**已完成时**，该顶点所代表的事件才能发生
#### 计算
| 名称       | 公式                    | 顺序       | 事件$v_k$，活动$a_i$                       |
| -------- | --------------------- | -------- | ------------------------------------- |
| 最早事件发生时间 | $ve(j)=\max(ve(i)+w)$ | 正向（拓扑序）  | 指从源点$v_1$到顶点$v_k$ 的最长路径长度             |
| 最迟事件发生时间 | $vl(i)=\min(vl(j)-w)$ | 逆向（逆拓扑序） | 不推迟整个工程完成时间的前提下，事件 $v_k$ 必须发生的最晚时间    |
| 最早开始时间   | $e=ve(i)$             | 直接取起点    | 该活动所对应弧的起点事件的最早发生时间                   |
| 最迟开始时间   | $l=vl(j)-w$           | 终点减边权    | 在不延误工程总工期的前提下，活动$a_i$最迟必须开始的时间        |
| 时间余量     | $d=l-e$               | 判断关键活动   | 活动$a_i$的最迟开始时间$l(i)$与其最早开始时间$e(i)$的差额 |
| 关键活动     | $e=l$ 或 $d=0$         |          |                                       |
| 关键路径     | 连续关键活动组成              |          |                                       |
