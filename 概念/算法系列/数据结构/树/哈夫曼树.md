## 编码
---
### 编码和解码
- **编码**：将信息转换成机器可处理形式。
- **解码**：编码的逆过程。

---
### 编码集分类
#### 按码长
| 类型   | 特点       |
| ---- | -------- |
| 定长编码 | 所有码字长度相同 |
| 变长编码 | 码长可不同    |
#### 按前缀性质
| 类型    | 特点             |
| ----- | -------------- |
| 前缀编码  | 任一码字都不是其他码字的前缀 |
| 非前缀编码 | 存在码字是其他码字的前缀   |

前缀编码：
- 可唯一解码
- 不需要分隔符
- 所有码字对应叶结点

---
### 定长编码
#### 定义
每个符号分配相同长度的码字
#### 码长
$$
l=\lceil\log_2n\rceil
$$
- n 为符号种类数
#### 特点
- 所有码字长度相同
- 叶结点位于同一层

---
### 编码长度计算
#### 加权路径长度 WPL
- $f_i$：频次
- $l_i$：编码长度
- WPL：全部数据占用的总位数
- 单位：bit
$$
WPL=\sum_{i=1}^nf_il_i
$$
#### 加权平均长度

$$
L=\frac{\sum_{i=1}^{n}f_il_i}{\sum_{i=1}^{n}f_i}
=\sum_{i=1}^{n}p_il_i
$$
- L：平均每个符号占用的位数
- 单位：`bit/symbol`
> 平均长度越小，编码效率越高

## 哈夫曼树
---
### 特点
- 带权二叉树
- 叶结点存放数据
- 内部结点权值 = 左右孩子权值之和
- 权值大 → 深度浅
- 权值小 → 深度深
### 构造过程
![[Pasted image 20260611172624.png]]
1. n 个叶结点构成森林
2. 选取权值最小的两棵树
3. 合并生成新结点
4. 新权值 = 左右子树权值之和
5. 删除原两树，将新树加入森林
6. 重复直到只剩一棵树
### 性质
- 初始结点最终都成为叶结点
- 权值小 → 路径长
- 权值大 → 路径短
- 新建结点数 = n-1
- 总结点数 = 2n-1
- 不存在度为 1 的结点
- 为严格二叉树
### WPL计算
#### 计算公式
$$WPL=\sum_{i=1}^{n}w_i\times l_i$$
其中：
- $w_i$：第 $i$ 个叶子结点的权值
- $l_i$：从根结点到该叶子结点的路径长度（边数）
- 所有叶子结点的$w_i\times l_i$​ 相加，就是带权路径长
#### 计算步骤
1. 找出所有叶子结点；
2. 求每个叶子到根的路径长度（经过几条边）；
3. 用权值 × 路径长度；
4. 全部加起来
#### 加权平均长度计算
$$L=\frac{WPL}{\sum_{i=1}^{n}w_i}$$
> 加权平均长度 = 带权路径长（WPL）÷ 权值总和

---
### 哈夫曼编码
#### 生成过程
1. 权值作为字符频度
2. 构造哈夫曼树
3. 左记 0，右记 1
4. 根到叶路径组成编码
#### 特点
- 变长编码
- 前缀编码
- 可唯一译码
- 所有二叉树中 WPL 最小
- 权值大 → 编码短
- 权值小 → 编码长

---
### 哈夫曼树不唯一
#### 原因
- 左右子树交换
- 相同权值结点合并顺序不同
#### 三不变
- WPL 不变
- 平均编码长度不变
- 最优性不变

---
### 定长编码与哈夫曼编码
| 项目     | 定长编码   | 哈夫曼编码  |
| ------ | ------ | ------ |
| 码长     | 相同     | 不同     |
| 类型     | 固定长度编码 | 可变长度编码 |
| 是否前缀编码 | 不一定    | 一定     |
| 平均长度   | 较大     | 最小     |
| 压缩效果   | 无      | 有      |
| 高频字符   | 无区别    | 编码短    |
| 低频字符   | 无区别    | 编码长    |
### 常考结论
- 哈夫曼树结点数 = 2n-1
- 哈夫曼树无度为 1 的结点
- 哈夫曼树是严格二叉树
- 哈夫曼编码一定是前缀编码
- 前缀编码一定能唯一译码
- WPL 最小，但树形不唯一
- 左右分支 0、1 可交换
- 权值越大，编码越短
- 权值越小，编码越长
- 哈夫曼树用于构造最优前缀编码
- 哈夫曼树不唯一，但 WPL 唯一
- 哈夫曼树的叶结点数为 n 时，总结点数为 2n-1
