# 作用

- **UI 组件** 的生命周期管理

- **状态管理**

- **跨组件通信**

- **日志记录**

- **性能优化**

- **权限管理**

在 ArkTS 中，装饰器**通常配合声明式 UI 框架**，用于管理 **State、提供自定义行为** 等

# **启用装饰器**

- 装饰器是 TypeScript 的实验性特性，需要在 `tsconfig.json` 中启用：

```
{
  "compilerOptions": {
    "experimentalDecorators": true}
}
```

然后你就可以在 TypeScript 代码中使用装饰器了。

# 常见装饰器

- `@Entry` ——描述文件为入口文件，页面

- `@State` —— 定义组件的状态变量

- `@Component` ——是一个组件

- `@Prop` —— 组件的输入属性

- `@Provide` / `@Consume` —— 跨组件共享数据

- `@Watch` —— 监听变量变化

- 自定义装饰器 —— 用于增强方法、类和属性的功能
