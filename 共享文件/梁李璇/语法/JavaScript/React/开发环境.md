# 使用create-react-app快速创建开发环境
```shell
npx create-react-app react_name
```
- npx：Node.js工具命令，查找并执行后续的包命令
- create-react-app：核心包（固定写法），用于创建React项目
- react_name：项目名称
# Vite（更流行）
Vite 创建应用和启动过程相对简单，与 CRA 的流程类似，但速度和性能更好。以下是 Vite 创建和启动一个 React 应用的步骤，以及 Vite 和 CRA 的一些主要区别：
### 1. 使用 Vite 创建和启动 React 应用
#### 安装 Vite 和创建项目
1. **创建项目：** 你可以通过以下命令来创建一个新的 Vite 项目。Vite 支持使用模板来初始化项目。
```shell
npm create vite@latest my-vite-app --template react
```
或者使用 yarn:
```shell
yarn create vite@latest my-vite-app --template react
```
2. **进入项目目录：**
```shell
cd my-vite-app
```
3. **安装依赖：** 在项目目录下运行以下命令来安装所有依赖：
```shell
npm install
```
或者：
```shell
yarn
```
#### 启动开发服务器
1. 启动开发服务器：
```shell
npm run dev
```
或者：
```shell
yarn dev
```
2. 访问开发服务器，默认情况下可以在浏览器中访问 `http://localhost:5173`。
### 2. Vite 和 CRA 的区别

|   |   |   |
|---|---|---|
|特性|Vite|CRA|
|**构建工具**|使用 Rollup（生产环境），开发时使用原生 ES 模块|使用 Webpack，开发和生产环境都一样|
|**启动速度**|快速启动，基于原生 ES 模块，开发时几乎即时热重载|启动较慢，Webpack 热重载慢|
|**热重载**|使用浏览器原生的 ES 模块支持，热重载更快|通过 Webpack Dev Server 实现热重载|
|**构建速度**|使用 Rollup，构建速度非常快|使用 Webpack，构建速度相对较慢|
|**配置**|默认配置就能开箱即用，支持按需自定义配置|配置相对复杂，需要更多的配置才能优化性能|
|**兼容性**|更适合现代浏览器，支持 ES 模块|支持较为广泛，但不如 Vite 高效|
