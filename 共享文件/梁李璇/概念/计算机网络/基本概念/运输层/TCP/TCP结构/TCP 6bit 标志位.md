
| TCP 标志位 | 含义     | 说明                                                              |
| ------- | ------ | --------------------------------------------------------------- |
| **ACK** | 确认应答   | ACK=1 时表示确认号有效；确认号表示**期望收到对方下一个字节的序号**，也就是说明 `x` 之前（含 x-1）都收到了。 |
| **SYN** | 同步序号   | 用于建立连接（三次握手中的前两步）。                                              |
| **FIN** | 结束发送   | 用于正常关闭连接（四次挥手）。                                                 |
| **RST** | 重置连接   | **立即终止连接**或拒绝无效连接，不走四次挥手，粗暴版断线。                                 |
| **CWR** | 拥塞窗口减少 | 用于明确拥塞提醒。发送方告诉对方 “我已经降低窗口了”。                                    |
| **ECE** | 显式拥塞通知 | 代表网络中出现拥塞，用于 ECN 机制下的拥塞报告。                                      |
| **PSH** | 推送数据   | 置位后要求接收方**立刻将数据交给上层应用**，不必在缓冲区中等待积累。                            |
| **URG** | 紧急数据   | 说明报文中有被上层标记为“紧急”的数据，需要优先处理。                                     |
