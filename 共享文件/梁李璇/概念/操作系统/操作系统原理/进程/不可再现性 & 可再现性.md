#### 不可再现性
- **定义**：多个线程或进程同时访问共享资源时，由于执行顺序的不确定性，程序的输出可能不同。
- **原因**：
    1. **线程调度不可预测**：操作系统调度线程的顺序可能每次都不一样。
    2. **竞态条件（Race Condition）**：两个线程同时修改同一变量，没有锁或同步机制时，结果依赖执行顺序。
    3. **外部环境变化**：I/O 延迟、网络延迟等也会影响程序运行顺序。
- **例子**：
```c
int x = 0;  
void thread1() { 
	x = 1; 
} 
void thread2() {
	x = 2; 
}  // 不加同步，每次运行 x 最终可能是 1 或 2

```    
- 这个就是典型的**不可再现性**。
---
#### 与可再现性的对比
- **可再现性（Reproducibility）**：相同操作、相同输入，每次得到相同结果。
- **不可再现性（Non-reproducibility）**：相同操作、相同输入，每次可能得到不同结果，通常是因为**不可控因素或并发冲突**。
