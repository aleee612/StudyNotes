## 中断流程
### 1. 现代操作系统使用虚拟内存
- 程序看到的是“虚拟地址”。
- 实际物理内存可能没有那么大。
- 所以一些数据暂时放在磁盘（swap / pagefile）。
### 2. 程序要访问某个页面
CPU 查页表 → 发现这个页面在内存中是“无效的”。
比如页表项里标记为：
- Present bit = 0 → 不在内存里
### 3. 缺页中断触发
CPU 直接叫操作系统过来处理：  
“欸，这一页呢？我要用啊。”
### 4. 操作系统做的事
- 暂停你正在运行的程序
- 找一个空闲物理页，或踢掉(换出)一个旧的
- 到磁盘里把该页面读进来
- 更新页表
- 恢复程序继续执行
这整个过程称为 **缺页中断处理流程（Page Fault Handling）**。
## 分类
- 内部异常（中断） -> 故障Fault（软件中断）
