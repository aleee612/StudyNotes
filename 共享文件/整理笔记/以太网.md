### 设备
#### 集线器
| **特性**   | **详细说明**                               |
| -------- | -------------------------------------- |
| **工作层次** | 物理层                                    |
| **拓扑结构** | 星形                                     |
| **冲突域**  | 整个集线器属于一个**冲突域**<br>同一时刻只能有一个设备发送数据    |
| **广播域**  | 整个集线器属于一个**广播域**                       |
| **通信模式** | **半双工**<br>设备不能同时发送和接收数据               |
| **安全性**  | 较低<br>因为数据是广播的，网络中的任何设备理论上都能抓取到发往他人的数据 |
**作用：**
- 将从接口接收到的比特向**所有**其他接口转发
- 中继：
	接收到信号后对其进行**整形**和**放大**
- 延长距离：
	通过转发有效克服电缆长度的限制，**扩大**网络的**覆盖范**围
**碰撞与解决：**
- 当某集线器同时从两个不同的接口接受到帧，将出现碰撞
- 生成该帧的节点必须重新传输该数据
#### 交换机
- 优于集线器，是无碰撞的，而且是名副其实的转发分组
- [[共享文件/整理笔记/链路层交换机|链路层交换机]]

### 以太网帧结构
| 字段       | 作用                                                                        |
| -------- | ------------------------------------------------------------------------- |
| **前同步码** | 前七个字节为10101010，最后一个字节为10101011<br>前七位作用：<br>	**唤醒**接受适配器，并将其时钟与发送方的**同步** |
| **目的地址** | 目的适配器的**MAC地址**<br>当收到为 广播地址 或者 本地MAC 则上交至网络层                             |
| **源地址**  | 发送方的适配器的MAC地址                                                             |
| **类型字段** | 允许以太网**复用**多种**网络层协议**，当字段值为0806（16进制）为ARP<br>**作用：** 与上一层的协议结合           |
| **数据字段** | IP数据报内容                                                                   |
| **CRC**  | [[差错检测和纠正#循环冗余检测CRC\|循环冗余检测]] -> FCS                                      |
