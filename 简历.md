## 项目经历

### 芙影视界平台（仿B站）

`SpringBoot` `SpringAI` `MybatisPlus` `MySQL` `Redis` `Elasticsearch` `Kafka` `Netty` `FFmpeg`

- **项目描述**：本项目是一个视频分享与互动交流平台，构建了一个高质量视频社区。支持实时通信交流、视频播放、视频上传、弹幕上传等功能。
- **项目职责**：
  1、认证授权：基于 SpringSecurity + JWT 构建了无状态登录体系。采用 RBAC 模型精细化控制功能权限。
  2、缓存优化：针对高频访问的用户、视频数据，使用 Redis 缓存进行读操作优化，将请求压力从 MySQL 转移，提升系统加载速度和 QPS。
  3、全文搜索：基于Elasticsearch实现了视频的关键词搜索视频功能，解决了传统MySQL模糊查询不够精确的问题。并结合Redis ZSet高效实现了热搜词的实时排行。
 
  4、实时通信：基于 Netty NIO 实现高性能 WebSocket 实时通信功能，支持多端同步、消息互通；并结合异步线程CompletableFuture编排业务逻辑，解决高并发下的消息阻塞问题。
 
   5、视频转码：视频上传集成了 FFmpeg 异步转码与 HLS 多码率切片，配合自定义线程池，实现了大文件的断点续传和秒传，转码后的视频异步上传至 OSS，实现了视频资源的快速分发，接口响应速度提升50%。
   
   6、AI视频总结：通过FFmpeg提取视频的音轨文件实现了从视频到MP3音频的转换，再集成星火大模型实现从MP3音频到文本内容的输出，最后通过文本内容进行视频总结。
   
   7、数据同步：使用 Canal 监听 MySQL binlog 实现数据实时同步至 Redis；通过死信队列保障同步失败后的重试机制，确保 MySQL 与 Redis 间数据最终一致性。
  
  8、消息队列：引入 Kafka 对播放统计、日志等高频写入请求进行削峰填谷，成功抵御每秒 3000 次突发请求冲击；使用 Guava 令牌桶算法实现服务层单机限流，保障核心服务 QPS 稳定在 1000。
 
 	9、推荐去重：构建Redis+RocksDB分层架构，引入布隆过滤器使存储空间压缩95%以上；通过时间轮异步批处理，将磁盘写入QPS降低90%，支撑海量用户的推荐视频去重。
  
 ### RAG智能商品推荐系统

`SpringBoot` `SpingAI` `MybatisPlus` `Redis` `MySQL` 

- **项目描述**：本项目基于SpringBoot与SpringAI构建了一款商品智能推荐系统，融合RAG、ToolCalling、MCP等技术，为用户提供个性化、多轮交互式的智能商品推荐服务。

- **项目职责**：

	1、集成了通义千问大模型，实现RAG检索和工具调用对话逻辑，并采用 SSE 流式响应机制输出文本内容。
  
  2、设计并落地 RAG (检索增强生成) 架构，利用 Redis Stack 构建商品向量数据库，通过语义检索精准召回商品信息，有效解决了大模型推荐中的“幻觉”问题。
  
  3、基于 MCP 协议集成网页抓取服务，赋予 AI 实时获取外部电商网站数据的能力，实现了基于实时信息的精准商品推荐。集成Function Calling，调用后端 服务的实时数据获取能力，实现了基于真实数据的精准推荐。
  
  4、通过集成Chat Memory 机制，实现了对多轮对话上下文的完整感知和管理，确保了用户交互的连贯性。
