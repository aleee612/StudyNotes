# 项目目录说明

本文档用于说明本项目各个文件夹和关键文件的作用，方便答辩、维护和后续开发。

## 根目录

`app.js`

后端服务入口文件。负责加载环境变量、创建 Express 应用、挂载后端接口路由、初始化数据库表，并启动后端服务。

`package.json`

后端项目配置文件。记录后端依赖、启动命令和演示数据脚本。

`package-lock.json`

后端依赖锁定文件。用于保证不同电脑安装依赖时版本一致。

`.env`

后端环境变量配置文件，例如 JWT 密钥、Token 过期时间等敏感配置。

`database.sqlite`

SQLite 数据库文件。系统中的用户、小组、项目、任务、好友、邀请等数据都存储在这里。

`软件工程答辩可能提问100问.txt`

答辩准备文档，整理了软件工程老师可能提问的问题和参考回答。

`项目目录说明.md`

当前文件，用于说明项目目录结构和文件含义。

## 后端目录

后端主要由 `config`、`router`、`controller`、`service`、`middleware`、`scripts` 组成。

### `config`

后端配置目录。

`config/db.js`

数据库连接和 SQLite 封装文件。负责连接 `database.sqlite`，开启外键约束，并封装 `execute`、`query`、`transaction` 等数据库操作方法。

### `middleware`

后端中间件目录。

`middleware/auth.js`

JWT 鉴权中间件。负责检查请求头中的 Token，验证登录身份，并把用户信息写入 `req.user`。

### `router`

后端路由目录。负责定义接口地址，并把请求分发给 controller。

`router/authRoutes.js`

登录、注册相关接口路由。

`router/projectRoutes.js`

项目相关接口路由，例如创建项目、查询项目、查询项目成员。

`router/taskRoutes.js`

任务相关接口路由，例如创建任务、查询任务、编辑任务、归档任务、恢复任务、删除任务。

`router/groupRoutes.js`

小组相关接口路由，例如创建小组、查询小组、邀请成员、处理小组邀请。

`router/friendRoutes.js`

好友相关接口路由，例如好友列表、发送好友申请、同意/拒绝好友申请。

`router/dashboardRoutes.js`

工作台统计接口路由，用于返回项目数、任务数、逾期任务、完成率等统计数据。

### `controller`

后端控制器目录。负责处理 HTTP 请求和响应，不直接写复杂业务逻辑。

`controller/authController.js`

处理用户注册和登录请求。

`controller/projectController.js`

处理项目创建、项目查询、项目成员查询等请求。

`controller/taskController.js`

处理任务创建、查询、更新、归档、恢复和删除请求。

`controller/groupController.js`

处理小组创建、成员邀请、成员移除、小组邀请处理等请求。

`controller/friendController.js`

处理好友列表、好友申请、同意/拒绝好友申请等请求。

`controller/dashboardController.js`

处理工作台统计数据请求。

### `service`

后端业务层和数据模型目录。这里是后端核心逻辑所在。

`service/authService.js`

用户认证业务逻辑，包括注册、密码加密、登录校验和 JWT 生成。

`service/userModel.js`

用户数据模型，负责用户表创建、用户创建、按用户名查询用户。

`service/projectService.js`

项目业务逻辑，例如创建项目、小组项目权限校验、项目成员同步。

`service/projectModel.js`

项目数据模型，负责 projects 和 project_members 表的创建、查询和写入。

`service/taskService.js`

任务业务逻辑，例如任务权限校验、状态校验、归档规则、负责人校验。

`service/taskModel.js`

任务数据模型，负责 tasks 表的创建、查询、更新、归档、恢复和删除。

`service/groupService.js`

小组业务逻辑，例如创建小组、邀请成员、同意/拒绝小组邀请、权限校验。

`service/groupModel.js`

小组数据模型，负责 groups 和 group_members 表的创建、查询和写入。

`service/friendService.js`

好友业务逻辑，例如发送好友申请、同意好友申请、删除好友。

`service/friendModel.js`

好友数据模型，负责 friends 和 friend_requests 表的创建、查询和写入。

`service/groupInvitationModel.js`

小组邀请数据模型，负责 group_invitations 表的创建、查询和状态更新。

`service/dashboardService.js`

工作台统计业务逻辑，负责统计项目、任务、逾期任务、完成率、最近任务等数据。

### `utils`

后端工具目录。

`utils/errorResponse.js`

统一错误响应工具。用于返回统一格式的错误信息，例如 `error`、`code`、`message`。

### `scripts`

脚本目录。

`scripts/seedDemo.js`

演示数据脚本。执行 `npm run seed:demo` 后，会创建 demo 用户、小组、项目、任务、好友和待处理申请，方便答辩展示。

## 前端目录

前端代码位于 `frontend` 文件夹中，使用 React + TypeScript + Vite。

### `frontend/package.json`

前端项目配置文件。记录前端依赖和脚本命令，例如 `npm run dev`、`npm run build`。

### `frontend/vite.config.ts`

Vite 配置文件。配置前端开发服务器端口 `5173`，并把 `/api` 请求代理到后端 `http://localhost:3000`。

### `frontend/index.html`

前端 HTML 入口文件。React 应用会挂载到其中的 `root` 节点。

### `frontend/src`

前端源码目录。

`frontend/src/main.tsx`

前端入口文件。负责渲染 React 应用，并引入全局样式。

`frontend/src/App.tsx`

前端主组件。负责整体页面布局、顶部导航、页面切换、项目创建弹窗和登录保护。

`frontend/src/styles.css`

全局 UI 样式文件。包括登录页、侧边栏、顶部导航、工作台、任务看板、小组管理、好友管理、弹窗、按钮、表单等所有主要界面样式。

## 前端组件目录

`frontend/src/components`

前端 UI 组件目录。

`DashboardOverview.tsx`

工作台页面组件。展示项目统计、任务统计、完成率圆环、逾期任务、最近任务和项目概览。

`Sidebar.tsx`

左侧项目导航组件。展示个人项目和小组项目，用于切换当前项目。

`TaskBoard.tsx`

任务看板组件。展示待办、进行中、已完成三列任务，支持新建任务、编辑任务、分配负责人、设置日期、归档、恢复和删除。

`GroupManager.tsx`

小组管理组件。展示小组列表、小组项目、小组成员、收到/发出的小组邀请，并支持管理员发送成员邀请。

`FriendManager.tsx`

好友管理组件。展示好友列表、发送好友申请、收到/发出的好友申请，并支持同意或拒绝好友申请。

`ProjectCreateModal.tsx`

新建项目弹窗组件。支持填写项目名称、描述、初始状态、所属小组、开始日期和截止日期。

## 前端 hooks 目录

`frontend/src/hooks`

前端自定义 Hook 目录。

`useAuth.tsx`

认证状态管理 Hook。负责登录、退出、保存用户信息、监听 Token 失效。

`useProjectTasks.ts`

项目任务数据 Hook。封装任务查询、新建、编辑、归档、恢复和删除操作。

## 前端 services 目录

`frontend/src/services`

前端接口封装目录。

`api.ts`

Axios 请求封装文件。统一设置 API 地址、请求 Token、401 处理，并导出 authAPI、projectAPI、taskAPI、groupAPI、friendAPI、dashboardAPI。

## 前端 types 目录

`frontend/src/types`

前端 TypeScript 类型定义目录。

`index.ts`

定义前端使用的数据类型，例如 User、Project、Task、Group、Friend、FriendRequest、GroupInvitation、DashboardOverview 等。

## node_modules 目录

`node_modules`

后端依赖安装目录，由 npm 自动生成，不需要手动修改。

`frontend/node_modules`

前端依赖安装目录，由 npm 自动生成，不需要手动修改。

## 前后端对应关系

用户登录注册：

- 前端：`frontend/src/pages/Login.tsx`
- 后端路由：`router/authRoutes.js`
- 后端控制器：`controller/authController.js`
- 后端业务：`service/authService.js`

工作台统计：

- 前端：`frontend/src/components/DashboardOverview.tsx`
- 后端路由：`router/dashboardRoutes.js`
- 后端控制器：`controller/dashboardController.js`
- 后端业务：`service/dashboardService.js`

项目管理：

- 前端：`Sidebar.tsx`、`ProjectCreateModal.tsx`
- 后端路由：`router/projectRoutes.js`
- 后端控制器：`controller/projectController.js`
- 后端业务：`service/projectService.js`、`service/projectModel.js`

任务管理：

- 前端：`TaskBoard.tsx`、`useProjectTasks.ts`
- 后端路由：`router/taskRoutes.js`
- 后端控制器：`controller/taskController.js`
- 后端业务：`service/taskService.js`、`service/taskModel.js`

小组协作：

- 前端：`GroupManager.tsx`
- 后端路由：`router/groupRoutes.js`
- 后端控制器：`controller/groupController.js`
- 后端业务：`service/groupService.js`、`service/groupModel.js`、`service/groupInvitationModel.js`

好友申请：

- 前端：`FriendManager.tsx`
- 后端路由：`router/friendRoutes.js`
- 后端控制器：`controller/friendController.js`
- 后端业务：`service/friendService.js`、`service/friendModel.js`

UI 样式：

- 全局样式集中在 `frontend/src/styles.css`
- 各组件主要负责结构和交互

数据库：

- 数据库文件是 `database.sqlite`
- 数据库连接封装在 `config/db.js`
- 各业务表创建和字段迁移分布在对应的 model 文件中

## 常用命令

后端启动：

```bash
npm start
```

生成演示数据：

```bash
npm run seed:demo
```

前端启动：

```bash
cd frontend
npm run dev
```

前端构建：

```bash
cd frontend
npm run build
```
